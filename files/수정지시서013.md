# SSaC codegen 수정 지시서 — gin 전환

**요청자**: fullend
**일자**: 2026-03-09
**우선순위**: 높음

## 배경

fullend 생성 백엔드를 프로덕션급으로 전환하기 위해 `net/http` → `gin` 프레임워크로 교체한다.
SSaC generator의 Go 코드 생성 템플릿을 gin 대상으로 변경해야 한다.

## 변경 범위

SSaC parser/validator는 변경 없음. **generator만 변경**.

## 수정 항목

### 1. 함수 시그니처

**현재** (`go_target.go:59`):
```go
func Login(w http.ResponseWriter, r *http.Request) {
func GetCourse(w http.ResponseWriter, r *http.Request, courseID int64) {
```

**변경**:
```go
func Login(c *gin.Context) {
func GetCourse(c *gin.Context) {
```

gin은 경로 파라미터를 `c.Param()`으로 꺼내므로 함수 시그니처에 path param 불필요.

### 2. 경로 파라미터 추출

**현재**: 함수 시그니처에 path param을 인자로 받음 (glue-gen의 라우터가 파싱해서 전달)

**변경**: 함수 본문 상단에서 `c.Param()` + 타입 변환 코드 생성
```go
courseIDStr := c.Param("CourseID")
courseID, err := strconv.ParseInt(courseIDStr, 10, 64)
if err != nil {
    c.JSON(http.StatusBadRequest, gin.H{"error": "invalid path parameter"})
    return
}
```

string 타입이면 변환 없이:
```go
slug := c.Param("Slug")
```

### 3. 요청 본문 파싱

**현재** (`go_templates.go` + `go_target.go`):
```go
// FormValue (1개 이하)
email := r.FormValue("Email")

// JSON body (2개 이상)
var req struct { ... }
if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
    http.Error(w, "invalid request body", http.StatusBadRequest)
    return
}
```

**변경**: gin의 `c.ShouldBindJSON()` 사용
```go
// JSON body (2개 이상 — 기존과 동일 조건)
var req struct { ... }
if err := c.ShouldBindJSON(&req); err != nil {
    c.JSON(http.StatusBadRequest, gin.H{"error": "invalid request body"})
    return
}
```

FormValue 케이스 (1개 이하):
```go
email := c.Query("Email")          // GET query param
email := c.PostForm("Email")       // POST form param
```

### 4. 에러 응답

**현재**: `http.Error(w, "메시지", statusCode)`

**변경**: `c.JSON(statusCode, gin.H{"error": "메시지"})` + `return`
```go
// 현재
http.Error(w, "User 조회 실패", http.StatusInternalServerError)
return

// 변경
c.JSON(http.StatusInternalServerError, gin.H{"error": "User 조회 실패"})
return
```

### 5. 성공 응답

**현재** (`response json` 템플릿):
```go
w.Header().Set("Content-Type", "application/json")
json.NewEncoder(w).Encode(map[string]interface{}{
    "user": user,
})
```

**변경**:
```go
c.JSON(http.StatusOK, gin.H{
    "user": user,
})
```

### 6. authorize 시퀀스

**현재**:
```go
allowed, err := authz.Check(currentUser, "create", "course", nil)
```

`currentUser` 참조 방식 변경 — glue-gen의 JWT 미들웨어가 `c.Set("currentUser", user)` 저장하므로:

**변경**:
```go
currentUser := c.MustGet("currentUser").(*model.CurrentUser)
allowed, err := authz.Check(currentUser, "create", "course", nil)
```

단, `currentUser` 추출 코드는 authorize 시퀀스에서 한 번만 생성. authorize가 없어도 `@param ... currentUser` 패턴이 있으면 동일하게 추출.

### 7. import 변경

**현재**: `"net/http"` 항상 포함

**변경**: `"net/http"` + `"github.com/gin-gonic/gin"` 항상 포함
- `"encoding/json"` — `response json`에서 더 이상 불필요 (`c.JSON` 사용). 단, JSON body 파싱 시 `c.ShouldBindJSON` 사용하므로 역시 불필요.
- `"encoding/json"` import 트리거 제거

### 8. QueryOpts 파싱

**현재**: glue-gen이 라우터에서 query param을 파싱

**변경**: 핸들러 함수 안에서 `c.Query()` 또는 `c.DefaultQuery()`로 직접 파싱
```go
opts := QueryOpts{}
if v := c.Query("limit"); v != "" {
    opts.Limit, _ = strconv.Atoi(v)
}
if v := c.Query("sort"); v != "" {
    opts.Sort = v
}
```

## 템플릿 변경 요약

| 템플릿 | 주요 변경 |
|--------|----------|
| `authorize` | `currentUser` → `c.MustGet("currentUser")` |
| `get` | 변경 없음 (모델 호출 동일) |
| `guard nil` | `http.Error` → `c.JSON` |
| `guard exists` | `http.Error` → `c.JSON` |
| `guard state` | `http.Error` → `c.JSON` |
| `post` | 변경 없음 (모델 호출 동일) |
| `put` | `http.Error` → `c.JSON` |
| `delete` | `http.Error` → `c.JSON` |
| `call_component` | `http.Error` → `c.JSON` |
| `call_func` | `http.Error` → `c.JSON` |
| `response json` | `json.NewEncoder` → `c.JSON(200, gin.H{...})` |

## 함수 시그니처 변경

| 현재 | 변경 |
|------|------|
| `func Name(w http.ResponseWriter, r *http.Request)` | `func Name(c *gin.Context)` |
| `func Name(w http.ResponseWriter, r *http.Request, id int64)` | `func Name(c *gin.Context)` + 본문에서 `c.Param` |

## 참고

- gin 공식: `github.com/gin-gonic/gin`
- fullend glue-gen도 동시에 gin 라우터 생성으로 변경 예정 (Phase 027)
- SSaC의 `(h *Handler)` 리시버 패턴은 glue-gen이 처리 — SSaC generator는 bare function 생성
