# 수정지시서 030 실행결과

## 근본 원인

SSaC Inputs는 `map[string]string`이라 순서가 없다. `buildArgsCodeFromInputs`가 key를 알파벳순으로 정렬했지만, 실제 모델 시그니처는 sqlc 쿼리의 `$1, $2, $3` 순서를 따른다.

예: `UPDATE gigs SET status = $1 WHERE id = $2` → 시그니처 `UpdateStatus(status, id)` 이지만, 알파벳순이면 `UpdateStatus(id, status)` — 인자 뒤바뀜.

## 해결: sqlc SQL 본문에서 파라미터 순서 추출 ✅

### 1. `validator/symbol.go` — `loadSqlcQueries` 수정 + `extractSqlcParams` 신규

`loadSqlcQueries`가 `-- name:` 줄뿐 아니라 SQL 본문도 파싱하여 `$N ↔ 컬럼명` 매핑을 추출, `MethodInfo.Params`에 저장.

**SQL 파싱 패턴 3가지:**

| 패턴 | 예시 | 추출 |
|---|---|---|
| INSERT | `(email, password_hash, name) VALUES ($1, $2, $3)` | 컬럼 목록 순서 |
| WHERE | `WHERE room_id = $1 AND end_at > $2` | `col = $N` 정규식 |
| SET+WHERE | `SET status = $1 WHERE id = $2` | `col = $N` 정규식, $N 순 정렬 |

컬럼명은 `strcase.ToGoPascal`로 `snake_case` → `PascalCase` 변환 (SSaC Inputs key와 매칭).

### 2. `generator/go_target.go` — `buildArgsCodeFromInputs` 시그니처 변경

```go
// before
func buildArgsCodeFromInputs(inputs map[string]string) string

// after
func buildArgsCodeFromInputs(inputs map[string]string, paramOrder []string) string
```

- `paramOrder` 있으면 → 그 순서대로 배치, 남은 키(query 등)는 마지막
- `paramOrder` 없으면 → 기존 알파벳순 fallback (query는 마지막)

### 3. `generator/go_target.go` — `lookupParamOrder` 신규

`buildTemplateData` 호출부에서 심볼 테이블의 `MethodInfo.Params`를 조회하여 `buildArgsCodeFromInputs`에 전달.

### 4. `generator/go_target.go` — `deriveInterfaces` 수정

models_gen.go 인터페이스 파라미터 순서도 `mi.Params` 참조. 호출 코드와 인터페이스의 인자 순서가 일치.

### 5. Phase029 query 특수 처리 유지

심볼 테이블이 없는 fallback 경로에서 `query` 값을 마지막으로 보내는 로직 유지. `paramOrder`가 있으면 `query`는 sqlc 파라미터에 없으므로 자연스럽게 마지막으로 밀림.

## 테스트 추가

### `validator/validator_test.go` — 5개 추가

- `TestExtractSqlcParamsInsert`: INSERT 컬럼 순서 추출
- `TestExtractSqlcParamsWhere`: WHERE 절 `$N` 순서 추출
- `TestExtractSqlcParamsUpdate`: SET+WHERE 절 순서 추출
- `TestExtractSqlcParamsNoParams`: 파라미터 없는 SELECT
- `TestExtractSqlcParamsSingleWhere`: 단일 WHERE 조건

### `generator/generator_test.go` — 2개 추가

- `TestGenerateArgsOrderMatchesSqlc`: sqlc Params 순서 → SQL 순서대로 인자 배치 확인
- `TestGenerateArgsOrderFallbackAlpha`: 심볼 테이블 없으면 알파벳순 fallback 확인
