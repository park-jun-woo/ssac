# 수정지시서 003: QueryOpts가 FindByID 등 비-List 메서드에 잘못 추가되는 문제

## 문제

SSaC 모델 인터페이스 생성(`GenerateModelInterfaces`)에서, `HasQueryOpts`가 호출 함수의 OpenAPI operation 기준으로 설정된다. 예를 들어:

- `GetCourse` 함수가 `x-pagination`을 가지고 있으면, 해당 함수에서 호출하는 **모든** 모델 메서드에 `opts QueryOpts`가 추가된다.
- `GetCourse`가 `User.FindByID(course.InstructorID)`를 호출하면, `UserModel.FindByID`에도 `opts QueryOpts`가 추가된다.

### 현재 생성 코드 (오류)

```go
// models_gen.go
type UserModel interface {
    FindByID(courseInstructorID int64, opts QueryOpts) (*User, error)  // ← opts 불필요
    FindByEmail(email string) (*User, error)
}
```

SSaC service codegen은 `User.FindByID(course.InstructorID)`로 호출 — args 1개.
모델 인터페이스는 args 2개(int64, QueryOpts) — **빌드 에러**.

### 올바른 생성 코드

```go
type UserModel interface {
    FindByID(courseInstructorID int64) (*User, error)  // opts 없음
    FindByEmail(email string) (*User, error)
}
```

## 원인

`go_target.go` line 857:
```go
if op, ok := st.Operations[usage.FuncName]; ok && op.HasQueryOpts() {
    dm.HasQueryOpts = true
}
```

`usage.FuncName`은 호출하는 SSaC 함수 이름(예: `GetCourse`)이다. `GetCourse`의 OpenAPI operation이 x-pagination을 가지면 `HasQueryOpts`가 true가 되어, `GetCourse` 안에서 호출되는 **모든** 모델 메서드에 opts가 추가된다.

실제로 opts가 필요한 것은 **List 계열 메서드**(결과가 `[]Type`)뿐이다.

## 수정

`HasQueryOpts`를 설정하는 조건에 **결과 타입이 슬라이스인 경우**만 해당하도록 제한:

```go
// 변경 전
if op, ok := st.Operations[usage.FuncName]; ok && op.HasQueryOpts() {
    dm.HasQueryOpts = true
}

// 변경 후
if op, ok := st.Operations[usage.FuncName]; ok && op.HasQueryOpts() {
    // opts는 List 메서드(결과가 slice)에만 추가
    if usage.Result != nil && strings.HasPrefix(usage.Result.Type, "[]") {
        dm.HasQueryOpts = true
    }
}
```

## 영향

- `User.FindByID` 등 비-List 메서드에서 불필요한 `opts QueryOpts` 파라미터 제거
- fullend dummy-lesson: `get_course.go`의 `User.FindByID(course.InstructorID)` 빌드 에러 해소

## 추가 수정: QueryOpts 암묵적 삽입 → 명시적 `query` 예약 소스

### 문제

현재 QueryOpts는 codegen이 **암묵적으로** 모델 호출에 `opts` 인자를 삽입한다. SSaC spec에는 `query`가 보이지 않으므로 "spec = source of truth" 원칙에 위배된다.

```go
// 현재 SSaC spec — query 파라미터가 보이지 않음
// @get []Reservation reservations = Reservation.ListByUserID(currentUser.ID)

// 생성 코드 — opts가 갑자기 나타남
reservations, total, err := reservationModel.ListByUserID(id, opts)
```

### 수정

`query`를 `request`, `currentUser`, `config`과 같은 **예약 소스**로 추가한다:

```go
// 수정 후 SSaC spec — query가 명시적
// @get []Reservation reservations = Reservation.ListByUserID(currentUser.ID, query)
```

- `query` → codegen에서 `QueryOpts` 변수로 변환 (자동 생성은 유지)
- validator: OpenAPI에 x-extensions 있는데 SSaC에 `query` 없으면 **ERROR**
- validator: OpenAPI에 x-extensions 없는데 `query` 쓰면 **ERROR**
- 모델 인터페이스: `query` 인자가 있는 메서드에만 `opts QueryOpts` 추가 (추론 불필요)

### 영향

- 기존 수정(slice 타입 추론)은 **대체됨**: `query` 명시 여부로 판단하므로 타입 추론 불필요
- 예약 소스 목록: `request`, `currentUser`, `config`, `query`
- 기존 dummy-study spec 업데이트 필요: List 메서드에 `query` 인자 추가
