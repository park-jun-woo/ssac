# 수정지시서 028: ErrStatus를 @empty, @exists, @state, @auth에 확장

## 배경

현재 `ErrStatus`(trailing HTTP 상태 코드)는 `@call`에만 지원된다.
그러나 실전 SaaS 개발에서 guard/state/auth 시퀀스의 기본 상태 코드가 도메인 의미와 맞지 않는 경우가 빈번하다.

예시 — Multi-tenant 워크플로우 SaaS (dummy-zenflow):
- 크레딧 부족 → `402 Payment Required` 반환이 필요하지만, `@empty`는 항상 `404`만 반환
- 구독 만료 → `403` 반환이 필요하지만 `@state` 전이 실패는 항상 `409`
- 인증 만료 vs 권한 부족 → `401`과 `403`을 구분해야 하지만 `@auth`는 항상 `403`

`@call`에 이미 `ErrStatus` 파싱 패턴이 존재하므로, 동일 패턴을 4개 시퀀스 타입에 확장한다.

## 문법 변경

| 타입 | 현재 | 확장 | 기본값 |
|---|---|---|---|
| `@empty` | `target "message"` | `target "message" 402` | 404 |
| `@exists` | `target "message"` | `target "message" 422` | 409 |
| `@state` | `diagramID {inputs} "transition" "message"` | `... "message" 422` | 409 |
| `@auth` | `"action" "resource" {inputs} "message"` | `... "message" 401` | 403 |

trailing 숫자가 없으면 기존 동작 유지. 있으면 100-599 범위 검증 후 `ErrStatus`에 저장.

## 변경 위치

### 1. `parser/types.go` — 주석 수정

```go
// 현재 (라인 74)
```

### 2. `parser/parser.go` — `splitTargetMessage` 반환값 확장

현재 (라인 612-620):
```go
func splitTargetMessage(s string) (string, string) {
	quoteIdx := strings.IndexByte(s, '"')
	if quoteIdx < 0 {
		return strings.TrimSpace(s), ""
	}
	target := strings.TrimSpace(s[:quoteIdx])
	msg, _ := extractQuoted(s[quoteIdx:])
	return target, msg
}
```

변경:
```go
func splitTargetMessage(s string) (string, string, string) {
	quoteIdx := strings.IndexByte(s, '"')
	if quoteIdx < 0 {
		return strings.TrimSpace(s), "", ""
	}
	target := strings.TrimSpace(s[:quoteIdx])
	msg, remainder := extractQuoted(s[quoteIdx:])
	return target, msg, strings.TrimSpace(remainder)
}
```

이 함수는 `parseGuard`에서만 호출되므로 영향 범위 없음.

### 3. `parser/parser.go` — `parseGuard` 수정

현재 (라인 287-295):
```go
func parseGuard(seqType, rest string) *Sequence {
	rest = strings.TrimSpace(rest)
	target, msg := splitTargetMessage(rest)
	return &Sequence{
		Type:    seqType,
		Target:  target,
		Message: msg,
	}
}
```

변경:
```go
func parseGuard(seqType, rest string) *Sequence {
	rest = strings.TrimSpace(rest)
	target, msg, remainder := splitTargetMessage(rest)
	seq := &Sequence{
		Type:    seqType,
		Target:  target,
		Message: msg,
	}
	if remainder != "" {
		if code, err := strconv.Atoi(remainder); err == nil && code >= 100 && code <= 599 {
			seq.ErrStatus = code
		}
	}
	return seq
}
```

```go
func TestParseEmptyErrStatus(t *testing.T) {
	src := `package service

// @empty orgWithCredits "Insufficient credits" 402
func ActivateWorkflow() {}
`
	sfs := parseTestFile(t, src)
	seq := sfs[0].Sequences[0]
	assertEqual(t, "Type", seq.Type, SeqEmpty)
	assertEqual(t, "Target", seq.Target, "orgWithCredits")
	assertEqual(t, "Message", seq.Message, "Insufficient credits")
	if seq.ErrStatus != 402 {
		t.Errorf("expected ErrStatus 402, got %d", seq.ErrStatus)
	}
}
```

### 기존 테스트 호환성

trailing 숫자 없는 기존 `@empty`, `@exists`, `@state`, `@auth` 테스트는 `ErrStatus == 0`으로 유지.
`splitTargetMessage`, `parseTwoQuoted` 시그니처 변경으로 기존 호출부(`parseGuard`, `parseState`)만 수정하며, 다른 호출부 없음.

## SSaC 작성 문법 변경

기존 문법에 optional trailing 숫자 추가. 기존 SSOT 파일 수정 불필요.

```go
// 기존 (그대로 동작)
// @empty user "User not found"
// @state gig {status: gig.Status} "PublishGig" "Cannot transition"
// @auth "PublishGig" "gig" {UserID: currentUser.ID, ResourceID: gig.ID} "Forbidden"

// 확장 (커스텀 상태 코드)
// @empty orgWithCredits "Insufficient credits" 402
// @exists duplicate "Already exists" 422
// @state workflow {status: workflow.Status} "ActivateWorkflow" "Payment required" 402
// @auth "Execute" "workflow" {UserID: currentUser.ID, ResourceID: w.ID} "Token expired" 401
```
