# 수정지시서 003 실행결과: QueryOpts 비-List 메서드 제외 + `query` 예약 소스

## 실행일: 2026-03-10

## 수정지시서 원안 대비 변경

원안은 slice 타입 추론으로 비-List 메서드를 제외하는 것이었으나, 검토 결과 근본적 설계 문제(암묵적 삽입)를 함께 해결:

| 항목 | 원안 | 실행 |
|---|---|---|
| 비-List 제외 | `[]Type` 추론 | `query` 명시 여부로 판단 (추론 불필요) |
| opts 삽입 방식 | codegen 암묵적 삽입 | SSaC spec에 `query` 명시 필수 |
| 교차 검증 | 없음 | OpenAPI x-extensions ↔ SSaC `query` 불일치 검증 |

## 변경 내용

### Phase 009: `query` 예약 소스

#### parser (변경 없음)

`query`는 bare arg로 파싱 → `Arg{Source: "query"}`. 기존 파서가 이미 처리함.

#### generator/go_target.go

- `argToCode`: `query` → `opts` 변환 추가
- `needsQueryOpts`: OpenAPI+isListMethod 추론 → args에 `query` 소스 있는지 확인으로 변경
- `buildTemplateData`: 암묵적 `QueryOpts auto-append for List methods` 블록 제거, `hasQueryArg` + `[]Type` 결과일 때 `HasTotal=true`
- `deriveInterfaces`: Operation 추론 → args에 `query`가 있으면 `HasQueryOpts`, query arg는 모델 파라미터에서 제외
- `buildTemplateData` 시그니처: `hasQO` 매개변수 제거 (불필요)
- `isListMethod`, `hasAnyQueryOpts` 함수 제거 (미사용)
- `hasQueryArg` 헬퍼 추가

#### validator/validator.go

- `reservedSources`에 `query` 추가
- `argVarRef`에 `query` 스킵 추가 (변수 흐름 검증에서 예약 소스 제외)
- `validateQueryUsage` 함수 추가:
  - SSaC에 `query` 있는데 OpenAPI에 x-extensions 없으면 **ERROR**
  - OpenAPI에 x-extensions 있는데 SSaC에 `query` 없으면 **WARNING**

#### generator/generator_test.go

- `TestGenerateModelInterfaceQueryOptsOnlyList` → `TestGenerateModelInterfaceQueryOptsExplicit`로 변경 (명시적 query arg 기반)
- `TestGenerateQueryArg` 추가: query arg → opts 변환, total 포함 확인

#### validator/validator_test.go

3개 테스트 추가:
- `TestValidateQueryUsageMissing`: OpenAPI에 x-extensions 있지만 SSaC에 query 없음 → WARNING
- `TestValidateQueryUsageNoOpenAPI`: SSaC에 query 있지만 OpenAPI에 x-extensions 없음 → ERROR
- `TestValidateQueryUsageMatch`: 양쪽 일치 → 에러 없음

#### specs/dummy-study/service/list_my_reservations.go

```go
// 변경 전
// @get []Reservation reservations = Reservation.ListByUserID(currentUser.ID)

// 변경 후
// @get []Reservation reservations = Reservation.ListByUserID(currentUser.ID, query)
```
