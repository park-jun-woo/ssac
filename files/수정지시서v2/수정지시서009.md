# 수정지시서 009: @call 반환 타입에 기본 타입 사용 시 validate 에러

## 배경

`@call`의 반환은 항상 Response struct이다. 그런데 현재 ssac validate는 `@call string accessToken = auth.IssueToken(...)` 처럼 기본 타입(`string`, `int`, `int64`, `bool` 등)을 반환 타입으로 지정해도 에러 없이 통과시킨다.

## 문제

ssac validate가 `@call` 반환 타입에 기본 타입이 오면 에러를 발생시키지 않는다.

## 기대 동작

`@call` 반환 타입이 기본 타입(`string`, `int`, `int64`, `bool`, `float64` 등)이면 validate ERROR:

```
[ERROR] @call 반환 타입에 기본 타입 "string"은 사용할 수 없습니다 — Response struct 타입을 지정하세요
```

올바른 사용:
```
// @call auth.IssueTokenResponse accessToken = auth.IssueToken({...})
```

## 영향 범위

ssac validator (파서/검증 단계)
