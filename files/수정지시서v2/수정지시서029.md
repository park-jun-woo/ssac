# 수정지시서 029: scanXxx 변수명 충돌 + ListByXxx 인자 순서 수정

## BUG001: scanXxx 변수명 충돌

### 증상
`fullend gen` → `go build` 시 `s redeclared in this block` 컴파일 에러.

### 재현
DDL에 `schedules` 테이블이 있을 때 생성되는 코드:

```go
func scanSchedule(s interface{ Scan(...interface{}) error }) (*Schedule, error) {
    var s Schedule  // ← 파라미터 s와 재선언 충돌
```

`Session`, `Setting`, `Subscription` 등 S로 시작하는 모든 모델에서 동일 발생.

### 원인
codegen이 모델 타입 첫글자를 소문자로 사용하는데(`Schedule` → `s`), scanner 파라미터도 `s`여서 충돌.

### 수정
scanner 파라미터를 `row`로 변경 (모든 scanXxx 함수에서 통일).

```go
// before
func scanSchedule(s interface{ Scan(...interface{}) error }) (*Schedule, error) {
    var s Schedule

// after
func scanSchedule(row interface{ Scan(...interface{}) error }) (*Schedule, error) {
    var s Schedule
    err := row.Scan(&s.ID, &s.ProjectID, ...)
```

## BUG002: ListByXxx pagination 인자 순서 뒤바뀜

### 증상
`fullend gen` → `go build` 시 `cannot use opts as int64 value in argument` 컴파일 에러.

### 재현
OpenAPI에 `x-pagination`이 있는 List 엔드포인트에서 생성되는 코드:

```go
// ssac-gen이 생성하는 핸들러 코드
h.ProjectModel.ListByUserID(opts, currentUser.ID)

// ssac-model이 생성하는 인터페이스
ListByUserID(userID int64, opts model.QueryOpts) ([]Project, error)
```

인자 순서가 반대.

### 원인
ssac-gen이 모델 호출 생성 시 QueryOpts를 첫 번째 인자로 배치하지만, ssac-model gen은 비즈니스 파라미터 먼저, QueryOpts 마지막으로 생성.

### 수정
ssac-gen에서 모델 호출 시 QueryOpts를 항상 마지막 인자로 배치.

```go
// before
h.ProjectModel.ListByUserID(opts, currentUser.ID)

// after
h.ProjectModel.ListByUserID(currentUser.ID, opts)
```
