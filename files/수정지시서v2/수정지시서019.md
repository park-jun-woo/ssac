# 수정지시서 019: @publish 시퀀스 + @subscribe 트리거 신설

## 배경

SSaC 서비스 함수는 현재 HTTP 요청만 트리거로 지원한다. 비동기 이벤트 발행/구독을 선언적으로 처리하기 위해 `@publish`(시퀀스)와 `@subscribe`(트리거)를 추가한다.

- `@publish` — 11번째 시퀀스 타입. 함수 내부에서 이벤트를 발행한다.
- `@subscribe` — 함수 레벨 트리거. 큐 이벤트 수신 시 함수를 실행한다.

설계 상세: `~/.clari/repos/fullend/files/queue.md`

## 변경 요청

### 1. @publish 시퀀스 타입 추가

문법:
```
@publish "topic" {payload} [{options}]
```

- `"topic"` — 이벤트 토픽명 (문자열 리터럴, 필수)
- `{payload}` — 메시지 페이로드 (필수, 기존 args 형식)
- `{options}` — 전송 옵션 (선택)

예시:
```go
package service

// @get Order order = Order.FindByID({ID: request.OrderID})
// @empty order "Order not found"
// @put Order.UpdateStatus({ID: order.ID, Status: "completed"})
// @publish "order.completed" {OrderID: order.ID, Email: order.Email, Amount: order.Amount}
// @response { order: order }
func CompleteOrder() {}
```

지연 발행:
```go
// @publish "cart.abandoned" {CartID: cart.ID, UserID: currentUser.ID} {delay: 1800}
```

옵션:

| 옵션 | 타입 | 설명 |
|---|---|---|
| `delay` | int (초) | 지연 발행 |
| `priority` | string | 우선순위 ("high", "normal", "low") |

### 2. @subscribe 트리거 추가

문법:
```
@subscribe "topic"
```

함수 본문의 첫 번째 어노테이션으로 선언. 이후 시퀀스는 기존과 동일.

예시:
```go
package service

// @subscribe "order.completed"
// @get Order order = Order.FindByID({ID: message.OrderID})
// @call mail.SendEmail({To: message.Email, Subject: "주문 완료", Body: "결제가 완료되었습니다."})
// @put Order.UpdateNotified({ID: order.ID, Notified: "true"})
func OnOrderCompleted() {}
```

### 3. 입력 변수 `message` 추가

HTTP 트리거에서 `request`를 쓰듯, 구독 트리거에서는 `message`를 쓴다.

| 트리거 | 입력 변수 |
|---|---|
| HTTP | `request` |
| Queue | `message` |

`message.OrderID`, `message.Email` 형태로 페이로드 필드에 접근.

### 4. 검증 규칙

| 규칙 | Level |
|---|---|
| `@subscribe` 함수에 `@response` 있음 | ERROR |
| `@subscribe` 함수에서 `request` 사용 | ERROR |
| `@subscribe`가 아닌 함수에서 `message` 사용 | ERROR |
| `@publish` topic이 빈 문자열 | ERROR |
| `@publish` payload 없음 | ERROR |

### 5. 파서 출력

`@publish` 파싱 결과:
```go
Sequence{
    Type:    "publish",
    Topic:   "order.completed",
    Inputs:  map[string]string{"OrderID": "order.ID", ...},
    Options: map[string]string{"delay": "1800"},  // 선택
}
```

`@subscribe` 파싱 결과 — `ServiceFunc`에 필드 추가:
```go
ServiceFunc{
    Name:      "OnOrderCompleted",
    Subscribe: &SubscribeInfo{Topic: "order.completed"},  // nil이면 HTTP 트리거
    Sequences: [...],
}
```

### 6. 코드젠

`@publish` → Go 코드:
```go
// @publish "order.completed" {OrderID: order.ID, Email: order.Email}
// ↓
queue.Publish(ctx, "order.completed", map[string]any{
    "OrderID": order.ID,
    "Email":   order.Email,
})
```

`@publish` 옵션 → Go 코드:
```go
// @publish "cart.abandoned" {CartID: cart.ID} {delay: 1800}
// ↓
queue.Publish(ctx, "cart.abandoned", map[string]any{
    "CartID": cart.ID,
}, queue.WithDelay(1800))
```

`@subscribe` 함수의 코드젠은 HTTP 트리거와 동일하되, `request` 대신 `message`를 사용한다. 핸들러 등록은 fullend gluegen이 담당.
