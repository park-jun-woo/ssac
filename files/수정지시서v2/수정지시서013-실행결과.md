# 수정지시서 013 실행결과: POST 핸들러 request body 필드 c.Query() 버그 수정

## 실행일: 2026-03-10

## 원인

`collectRequestParams()`에서 JSON body 사용 기준이 `len(rawParams) >= 2`만 체크. POST/PUT 핸들러에서 path param 제외 후 request 필드가 1개만 남으면 `c.Query()` fallback.

## 변경 내용

### generator/go_target.go

`collectRequestParams()`에 POST/PUT 시퀀스 감지 추가:

```go
hasBodySeq := false
for _, seq := range seqs {
    if seq.Type == parser.SeqPost || seq.Type == parser.SeqPut {
        hasBodySeq = true
        break
    }
}
if (st != nil && len(rawParams) >= 2) || (hasBodySeq && len(rawParams) >= 1) {
    return buildJSONBodyParams(rawParams)
}
```

POST/PUT이 있으면 request 필드 1개라도 `ShouldBindJSON`으로 처리.

### generator/generator_test.go

`TestGeneratePostBodySingleField` 추가 — SubmitProposal 패턴 검증:
- `@get`에서 `request.ID` → path param
- `@post`에서 `request.BidAmount` → JSON body (1개 필드)
- `ShouldBindJSON(&req)` 사용 확인, `c.Query("BidAmount")` 미사용 확인
