# 수정지시서 008: @call result 단일 필드 자동 추출

## 배경

`@call string hashedPassword = auth.HashPassword({Password: request.Password})` 선언 시,
ssac-gen이 `hashedPassword, err := auth.HashPassword(auth.HashPasswordRequest{...})`를 생성한다.

`hashedPassword`의 실제 타입은 `HashPasswordResponse` struct이지만, SSaC에서 result 타입을 `string`으로 선언했다.
이후 `@post User.Create(request.Email, hashedPassword, ...)` 에서 `hashedPassword`를 `string`으로 사용하면 컴파일 에러가 발생한다.

```
cannot use hashedPassword (variable of struct type auth.HashPasswordResponse) as string value
```

## 현재 동작

```go
// SSaC
// @call string hashedPassword = auth.HashPassword({Password: request.Password})

// 생성 코드
hashedPassword, err := auth.HashPassword(auth.HashPasswordRequest{Password: password})
// hashedPassword = HashPasswordResponse{HashedPassword: "..."}  ← struct 전체
```

## 기대 동작

result 타입이 primitive (`string`, `int64` 등)이고 Response struct에 해당 타입 필드가 정확히 1개이면,
ssac-gen이 자동으로 필드를 추출해야 한다.

```go
// 생성 코드
hashedPasswordResp, err := auth.HashPassword(auth.HashPasswordRequest{Password: password})
hashedPassword := hashedPasswordResp.HashedPassword
```

또 다른 예시: `@call string accessToken = auth.IssueToken({...})`
```go
// 현재 (버그)
accessToken, err := auth.IssueToken(auth.IssueTokenRequest{...})
// accessToken = IssueTokenResponse{AccessToken: "jwt..."} ← struct

// 기대
accessTokenResp, err := auth.IssueToken(auth.IssueTokenRequest{...})
accessToken := accessTokenResp.AccessToken
```

## 추출 규칙

1. `@call` result에 타입이 명시되어 있다 (`string`, `int64`, `bool` 등 primitive)
2. Func spec Response struct에서 해당 타입의 필드를 찾는다
3. 필드가 정확히 1개면 → `resp.FieldName`으로 추출
4. 필드가 0개 또는 2개 이상이면 → 에러 (ambiguous)

## 수정 위치

### generator/go_target.go — @call 코드 생성

`@call` result가 있을 때:
- 현재: `varName, err := pkg.Func(pkg.FuncRequest{...})`
- 변경: `varNameResp, err := pkg.Func(pkg.FuncRequest{...})` + `varName := varNameResp.FieldName`

result 타입이 없거나 struct 타입이면 기존 동작 유지.

### 필요 정보

ssac-gen이 추출할 필드명을 알려면 Func spec의 Response struct 정보가 필요하다.
현재 ssac-gen은 Func spec을 모른다 → fullend가 ssac-gen에 추가 정보를 전달하거나,
ssac-gen 단독으로 해결할 방법이 필요하다.

**방안 A**: ssac-gen이 result 타입(`string`)과 변수명(`hashedPassword`)을 보고, Response 필드명을 PascalCase(hashedPassword → HashedPassword)로 추론.
**방안 B**: fullend가 ssac-gen 호출 시 Func spec 정보를 전달.

방안 A가 ssac 단독 해결이므로 더 간단하다.
