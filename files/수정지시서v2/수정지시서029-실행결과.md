## BUG002: ListByXxx pagination 인자 순서 수정 ✅

### 원인

`buildArgsCodeFromInputs`가 key를 알파벳순 정렬하여 `Opts`(`query`)가 `UserID`보다 앞에 배치됨.

```go
// before (알파벳순: Opts < UserID)
reservationModel.ListByUserID(opts, currentUser.ID)

// after (query를 마지막으로)
reservationModel.ListByUserID(currentUser.ID, opts)
```

### 변경 내용

#### `generator/go_target.go` — `buildArgsCodeFromInputs`

`query` 값을 가진 키를 분리하여 정렬된 키 목록의 마지막에 배치:

```go
keys := make([]string, 0, len(inputs))
var queryKey string
for k := range inputs {
    if inputs[k] == "query" {
        queryKey = k
    } else {
        keys = append(keys, k)
    }
}
sort.Strings(keys)
if queryKey != "" {
    keys = append(keys, queryKey)
}
```

#### `generator/generator_test.go`

`TestGenerateWithQueryOpts` assertion을 올바른 인자 순서로 수정:

```go
// before
assertContains(t, code, `reservationModel.ListByUserID(opts, currentUser.ID)`)

// after
assertContains(t, code, `reservationModel.ListByUserID(currentUser.ID, opts)`)
```
