# 수정지시서 025: @request struct 필드 unexported → PascalCase 전환

## 현재 문제

`buildJSONBodyParams()` (generator/go_target.go:669)에서 request struct 필드를 `rp.name` 그대로(소문자/snake_case) 생성.

```go
// line 669
buf.WriteString(fmt.Sprintf("\t\t%s %s `json:\"%s\"`\n", rp.name, rp.goType, rp.name))
```

`rp.name`은 SSaC `request.field`의 필드명 = OpenAPI 스키마의 필드명으로, `email`, `bid_amount` 등 소문자/snake_case.

생성 결과:
```go
var req struct {
    email      string `json:"email"`       // unexported → JSON 바인딩 불가
    bid_amount int32  `json:"bid_amount"`   // snake_case → Go 컨벤션 위반 + unexported
}
```

Go의 `encoding/json`은 exported 필드만 언마셜링한다. 현재 코드는 모든 request 필드가 zero value.

## 변경 목표

struct 필드를 Go PascalCase(exported)로, json tag는 원래 이름 유지.

```go
var req struct {
    Email     string `json:"email"`
    BidAmount int32  `json:"bid_amount"`
}
email := req.Email
bidAmount := req.BidAmount
```

## 변경 내용

### generator/generator.go — toPascalCase() 함수 추가

`iancoleman/strcase` 패턴 기반. delimiter(`_`, `-`, ` `) 분리 → 각 워드 capitalize → Go initialism 적용.
기존 `commonInitialisms` 맵(generator.go:57)을 그대로 활용.

```go
// toPascalCase converts snake_case/kebab-case/space-separated to Go PascalCase.
// Handles Go common initialisms: "id" → "ID", "url" → "URL", "smtp" → "SMTP"
// Examples: "email" → "Email", "bid_amount" → "BidAmount", "client_id" → "ClientID",
//           "smtp_host" → "SMTPHost", "user_api_key" → "UserAPIKey"
func toPascalCase(s string) string {
    if s == "" {
        return s
    }
    var result strings.Builder
    capNext := true
    wordStart := 0
    runes := []rune(s)

    for i := 0; i <= len(runes); i++ {
        isDelim := i == len(runes) || runes[i] == '_' || runes[i] == '-' || runes[i] == ' '
        if isDelim {
            if i > wordStart {
                word := string(runes[wordStart:i])
                upper := strings.ToUpper(word)
                if commonInitialisms[upper] {
                    result.WriteString(upper)
                } else if capNext {
                    result.WriteString(strings.ToUpper(word[:1]) + strings.ToLower(word[1:]))
                } else {
                    result.WriteString(strings.ToLower(word))
                }
                capNext = true
            }
            wordStart = i + 1
            continue
        }
    }
    return result.String()
}
```

참고: [github.com/iancoleman/strcase](https://github.com/iancoleman/strcase), [github.com/ettle/strcase](https://github.com/ettle/strcase)

변환 예:
| 입력 | 결과 |
|---|---|
| `email` | `Email` |
| `bid_amount` | `BidAmount` |
| `password_hash` | `PasswordHash` |
| `id` | `ID` |
| `client_id` | `ClientID` |
| `video_url` | `VideoURL` |
| `created_at` | `CreatedAt` |

### generator/go_target.go — buildJSONBodyParams()

#### Line 669: struct 필드명 toPascalCase 적용

```go
// 변경 전
buf.WriteString(fmt.Sprintf("\t\t%s %s `json:\"%s\"`\n", rp.name, rp.goType, rp.name))

// 변경 후
buf.WriteString(fmt.Sprintf("\t\t%s %s `json:\"%s\"`\n", toPascalCase(rp.name), rp.goType, rp.name))
```

#### Line 678: req 필드 참조도 toPascalCase 적용

```go
// 변경 전
buf.WriteString(fmt.Sprintf("\t%s := req.%s\n", varName, rp.name))

// 변경 후
buf.WriteString(fmt.Sprintf("\t%s := req.%s\n", varName, toPascalCase(rp.name)))
```

`varName`은 `lcFirst(rp.name)`으로 이미 camelCase이므로 변경 불필요. 단, snake_case 입력 시 `lcFirst("bid_amount")` → `"bid_amount"` (그대로). 이것도 camelCase로 변환해야 한다면:

```go
// line 677도 변경
varName := lcFirst(toPascalCase(rp.name))
```

이렇게 하면 `bid_amount` → `toPascalCase` → `BidAmount` → `lcFirst` → `bidAmount`.
