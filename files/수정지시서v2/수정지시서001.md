# 수정지시서 001: @get/@post 0-arg 허용

## 문제

`validator.go`의 `validateRequiredFields`에서 `@get`, `@post`에 `len(seq.Args) == 0`이면 `"Args 누락"` ERROR를 반환한다.

```go
case parser.SeqGet, parser.SeqPost:
    // ...
    if len(seq.Args) == 0 {
        errs = append(errs, ctx.err("@"+seq.Type, "Args 누락"))
    }
```

그러나 비즈니스 필터 없이 전체 조회하는 케이스는 정당하게 존재한다.

```go
// @get []Course courses = Course.List()
```

## 근거

- SSaC arg는 **비즈니스 로직 인자**이다 (CourseID, UserID 등)
- 페이지네이션/정렬/필터는 OpenAPI `x-pagination`, `x-sort`, `x-filter`로 선언되며, 코드젠 시 프레임워크 레이어에서 처리된다
- sqlc 쿼리에도 파라미터 없는 조회가 존재한다: `SELECT * FROM courses WHERE published = TRUE`
- 비즈니스 arg와 인프라 관심사(pagination)를 혼동하면 모델 인터페이스 시그니처가 오염된다

## 영향

- fullend dummy-lesson의 `ListCourses`, 및 유사한 전체 조회 서비스 함수가 validate를 통과하지 못한다

## 수정 방안

`validator.go` `validateRequiredFields`에서 `@get`, `@post`의 Args 필수 검증을 제거한다. `@put`, `@delete`도 동일하게 0-arg를 허용한다.

```go
// 변경 전
case parser.SeqGet, parser.SeqPost:
    if seq.Model == "" {
        errs = append(errs, ctx.err("@"+seq.Type, "Model 누락"))
    }
    if seq.Result == nil {
        errs = append(errs, ctx.err("@"+seq.Type, "Result 누락"))
    }
    if len(seq.Args) == 0 {
        errs = append(errs, ctx.err("@"+seq.Type, "Args 누락"))
    }

// 변경 후
case parser.SeqGet, parser.SeqPost:
    if seq.Model == "" {
        errs = append(errs, ctx.err("@"+seq.Type, "Model 누락"))
    }
    if seq.Result == nil {
        errs = append(errs, ctx.err("@"+seq.Type, "Result 누락"))
    }
    // Args는 0개 허용 (비즈니스 필터 없는 전체 조회)
```

`@put`, `@delete`도 동일:

```go
// 변경 전
case parser.SeqPut, parser.SeqDelete:
    if seq.Model == "" {
        errs = append(errs, ctx.err("@"+seq.Type, "Model 누락"))
    }
    if len(seq.Args) == 0 {
        errs = append(errs, ctx.err("@"+seq.Type, "Args 누락"))
    }

// 변경 후
case parser.SeqPut, parser.SeqDelete:
    if seq.Model == "" {
        errs = append(errs, ctx.err("@"+seq.Type, "Model 누락"))
    }
    // Args는 0개 허용
```

## 변경 파일

| 파일 | 변경 |
|------|------|
| `validator/validator.go` | `@get`/`@post`/`@put`/`@delete`의 Args 필수 검증 제거 |
| `validator/validator_test.go` | Args 누락 테스트 케이스가 있다면 제거 또는 수정 |
