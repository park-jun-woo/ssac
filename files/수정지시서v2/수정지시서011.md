# 수정지시서 011: @call 결과 변수 필드는 역참조하지 않아야 함

## 배경

수정지시서010에서 모델 필드 참조 시 `*` 역참조를 추가했다.
그런데 `@call` 결과 변수(Response struct)의 필드까지 역참조되고 있다.

Response struct 필드는 포인터가 아닌 일반 타입이므로 역참조하면 컴파일 에러.

## 문제

```go
// SSaC
// @call auth.HashPasswordResponse hashedPassword = auth.HashPassword({Password: request.Password})
// @post User user = User.Create({PasswordHash: hashedPassword.HashedPassword, ...})

// 현재 생성 코드 (잘못됨)
h.UserModel.Create(email, name, *hashedPassword.HashedPassword, role)
//                                ^ 역참조 불필요 — string 타입

// 올바른 생성 코드
h.UserModel.Create(email, name, hashedPassword.HashedPassword, role)
```

## 기대 동작

`inputValueToCode`에서 `source.field` 형태를 역참조할 때, source가 `@call` 결과 변수이면 역참조하지 않는다.

역참조 대상: `@get`, `@post`, `@put` 등 DB 모델 결과 변수의 필드만
역참조 제외: `@call` 결과 변수, `currentUser`, `config`, `request`

## 판단 기준

source 변수가 어떤 시퀀스에서 왔는지 확인:
- `@get`, `@post`, `@put` → 모델 결과 → 포인터 필드 → `*` 역참조
- `@call` → Response struct → 일반 필드 → 역참조 안 함

## 영향 범위

ssac generator — `inputValueToCode()` 함수
