# 수정지시서 010: nullable 컬럼 필드 사용 시 포인터 역참조 코드 생성

## 배경

fullend gluegen이 DDL에서 NOT NULL이 없는 컬럼을 `*int64`, `*string` 등 포인터 타입으로 생성하도록 수정되었다.

ssac-gen이 모델 필드를 참조하는 코드를 생성할 때, 해당 필드가 포인터 타입이면 역참조(`*field`)해서 사용해야 한다.

## 문제

현재 ssac-gen은 모델 필드를 그대로 사용한다:

```go
// 현재 생성 코드
billing.ReleaseFunds(billing.ReleaseFundsRequest{FreelancerID: gig.FreelancerID})
```

`gig.FreelancerID`가 `*int64`이므로 컴파일 에러:
```
cannot use gig.FreelancerID (variable of type *int64) as int64 value in struct literal
```

## 기대 동작

ssac-gen이 DDL(또는 SymbolTable)을 참조하여 nullable 컬럼 필드를 사용할 때 역참조 코드를 생성한다:

```go
// 기대 생성 코드
billing.ReleaseFunds(billing.ReleaseFundsRequest{FreelancerID: *gig.FreelancerID})
```

## 판단 기준

DDL 컬럼에 NOT NULL이 없고 PRIMARY KEY도 아니면 nullable → 포인터 타입 → 역참조 필요.

SymbolTable에 이미 DDL 정보가 있으므로, 코드 생성 시 해당 컬럼의 nullable 여부를 확인할 수 있어야 한다.

## 영향 범위

ssac generator — 모델 필드를 struct literal에 넣는 코드 생성 부분
