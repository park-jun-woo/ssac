# 수정지시서 017: .ssac 확장자 도입 + 패키지 모델 파라미터 매칭 검증

## 배경

### 문제 1: 서비스 파일 Go 빌드 충돌

SSaC 서비스 파일은 Go 문법을 사용하지만 실제 Go 코드가 아님 (주석으로 로직 선언, 함수 본체 비어있음). `.go` 확장자 때문에 Go 컴파일러가 빌드 대상으로 인식하여 `imported and not used` 에러 발생.

**해결: `.ssac` 확장자 도입.** Go 컴파일러는 `.ssac` 파일을 무시. SSaC 파서는 `go/parser.ParseFile()`이 확장자와 무관하게 동작하므로 정상 파싱.

### 문제 2: 파라미터 매칭 미검증

수정지시서 016에서 패키지 접두사 @model의 Go interface 로드 + 메서드 존재 검증이 구현됨.
현재는 메서드 존재만 확인하고, SSaC 파라미터명과 interface 파라미터명의 일치 여부는 검증하지 않음.

```go
// interface: Get(ctx context.Context, key string) (string, error)
// SSaC:     session.Session.Get({token: request.Token})  ← "token"은 interface에 없음 — 현재 에러 안 남
```

## 요청 사항

### 1. .ssac 확장자 도입

SSaC 서비스 파일의 확장자를 `.go` → `.ssac`으로 변경.

**변경 전:**
```
specs/my-project/service/auth/login.go
specs/my-project/service/auth/register.go
```

**변경 후:**
```
specs/my-project/service/auth/login.ssac
specs/my-project/service/auth/register.ssac
```

**파서 변경:**
- SSaC 파서가 서비스 디렉토리를 스캔할 때 `.ssac` 파일을 대상으로 함
- `go/parser.ParseFile()`은 확장자와 무관하게 Go 문법을 파싱하므로 내부 로직 변경 없음
- `.go` 파일은 더 이상 서비스 파일로 인식하지 않음

### 2. SSaC 파라미터명 ↔ Go interface 파라미터명 매칭 검증

패키지 접두사 @model의 SSaC 파라미터 키가 Go interface 메서드 파라미터명과 정확히 일치하는지 검증.

**규칙:**
- interface 파라미터에서 `context.Context` 타입은 제외 (Go 언어 관례, 프레임워크가 제공)
- 나머지 파라미터명과 SSaC `{key: value}` 키를 비교
- SSaC에 없는 interface 파라미터 → ERROR
- SSaC에 있지만 interface에 없는 파라미터 → ERROR

**interface:**
```go
// pkg/session/session.go
type SessionModel interface {
    Set(ctx context.Context, key string, value any, ttl time.Duration) error
    Get(ctx context.Context, key string) (string, error)
    Delete(ctx context.Context, key string) error
}
```

**SSaC 올바른 사용 (login.ssac):**
```go
package service

import "github.com/geul-org/fullend/pkg/session"

// @get Session session = session.Session.Get({key: request.Token})
func GetSession() {}
```

**SSaC 에러 — 파라미터명 불일치 (login.ssac):**
```go
package service

import "github.com/geul-org/fullend/pkg/session"

// @get Session session = session.Session.Get({token: request.Token})
func GetSession() {}
// ❌ ERROR: "token" not in interface (expected: key)
```

**SSaC 에러 — 파라미터 누락 (create_session.ssac):**
```go
package service

import "github.com/geul-org/fullend/pkg/session"

// @post Session result = session.Session.Set({key: userID, value: userData})
func CreateSession() {}
// ❌ ERROR: missing "ttl"
```

### 3. ERROR 메시지 형식

```
[ERROR] SSaC ↔ Interface: <funcName> — @model <pkg>.<Model>.<Method> 파라미터 불일치. SSaC에 %q가 있지만 interface에 없습니다. interface 파라미터: [%s]
[ERROR] SSaC ↔ Interface: <funcName> — @model <pkg>.<Model>.<Method> 파라미터 누락. interface에 %q가 필요하지만 SSaC에 없습니다. SSaC 파라미터: [%s]
```

### 4. context.Context 스킵 규칙

- interface 파라미터 타입이 `context.Context`이면 매칭 대상에서 제외
- 이는 Go 언어 보편 관례 (gin, echo, net/http 등 프레임워크 무관)
- interface에 `ctx context.Context`가 없으면 검증하지 않음 (사용자 선택)

### 5. 적용 범위

- `.ssac` 확장자: 모든 SSaC 서비스 파일
- 파라미터 매칭: 패키지 접두사 @model만 해당 (`seq.Package != ""`)
- DDL 모델 (접두사 없음)은 기존 방식 유지 (fullend crosscheck 담당)
