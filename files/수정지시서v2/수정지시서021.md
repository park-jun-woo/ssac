# 수정지시서 021: @response 단일 행 구조체 형식 파싱 지원

## 문제

`@response { field: var }` 를 한 줄로 작성하면 파서가 `{ field: var }` 전체를 변수명으로 인식하여 "변수가 아직 선언되지 않았습니다" 에러를 발생시킴.

## 원인

`parser/parser.go` `parseLine()` 함수 (162-182행):

```go
if trimmed == "{" {
    return nil, true, nil  // multiline 모드 시작
}
if trimmed != "" && trimmed != "{" {
    // trimmed가 "{ user: user }" 일 때 — 변수명으로 저장됨
    return &Sequence{Type: SeqResponse, Target: trimmed, ...}, false, nil
}
```

- `trimmed == "{"` 만 체크하므로, `"{ user: user }"` (한 줄 형식)은 multiline으로 인식되지 않음
- shorthand 분기에서 `Target`에 `"{ user: user }"` 전체가 저장됨
- 이후 validator가 이를 변수명으로 검증 → 에러

## 수정 제안

`parseLine()`에서 `trimmed`가 `{`로 시작하고 `}`로 끝나면 단일 행 구조체로 파싱:

```go
// @response { field: var, ... } — single-line struct
if strings.HasPrefix(trimmed, "{") && strings.HasSuffix(trimmed, "}") {
    inner := strings.TrimSuffix(strings.TrimPrefix(trimmed, "{"), "}")
    fields, err := parseResponseFields(inner)
    if err != nil {
        return nil, false, err
    }
    return &Sequence{
        Type:         SeqResponse,
        Fields:       fields,
        SuppressWarn: suppressWarn,
    }, false, nil
}
```

이 분기를 `trimmed == "{"` 체크 앞에 추가하거나, `trimmed != "{"` shorthand 분기 이전에 추가.

## 영향 범위

- `parser/parser.go`: `parseLine()` 함수
- 테스트 추가: `parser/parser_test.go`에 단일 행 `@response { field: var }` 테스트 케이스
