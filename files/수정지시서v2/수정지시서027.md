# 수정지시서 027: @auth 코드젠 Role 자동 추가 + validator inputs 참조 검증

## 배경

fullend Phase 030에서 `pkg/authz`의 `CheckRequest`에 `Role` 필드를 추가한다.
OPA Rego 정책이 `input.claims.role`을 참조하므로, SSaC `@auth` 코드젠이 `Role: currentUser.Role`을 자동으로 생성해야 한다.

또한 현재 SSaC validator는 `@auth` inputs의 개별 필드를 전혀 검증하지 않는다. 존재하지 않는 변수를 참조해도 통과한다. 이를 보완한다.

## Part 1: 코드젠 — Role 자동 추가

### 현재 동작

SSaC `@auth` 코드젠 결과:
```go
authz.Check(authz.CheckRequest{Action: "PublishGig", Resource: "gig", ResourceID: gig.ID, UserID: currentUser.ID})
```

`Role` 필드가 없어서 OPA `input.claims.role` 검사가 항상 빈값으로 실패한다.

### 기대 동작

```go
authz.Check(authz.CheckRequest{Action: "PublishGig", Resource: "gig", ResourceID: gig.ID, UserID: currentUser.ID, Role: currentUser.Role})
```

`@auth` 시퀀스의 Inputs에 `UserID: currentUser.ID`가 있으면(= currentUser를 사용하면), `Role: currentUser.Role`을 **자동으로** 함께 생성한다.

### 변경 위치

#### `generator/go_target.go` — auth 처리 분기

`@auth` 시퀀스의 InputFields 생성 시, Inputs map에 `UserID` 키가 있고 그 값이 `currentUser.`로 시작하면 `Role: currentUser.Role` 필드를 자동 추가한다.

**조건**: `seq.Type == parser.SeqAuth` && inputs에 `currentUser.*` 참조 존재
**추가**: `Role: currentUser.Role`

SSaC 작성자가 명시적으로 `Role`을 inputs에 쓸 필요 없이, `@auth`이고 currentUser를 사용하면 자동으로 붙는다.

#### `generator/go_templates.go`

템플릿 변경 없음. `{{.InputFields}}`에 `Role: currentUser.Role`이 포함되어 나옴.

### 테스트

- `TestGenerateAuthCallStyle` (라인 686~): `assertContains(t, code, "Role: currentUser.Role")` 추가
- `TestGenerateAuth` (라인 97~): 기존 `@auth` 테스트에 Role 검증 추가
- currentUser 없는 `@auth` 케이스 (`TestGenerateAuthNoCurrentUser`): Role이 생성되지 않음을 확인

## Part 2: validator — inputs 변수 참조 검증

### 현재 문제

`@auth`, `@state`, `@call` 등 모든 시퀀스의 inputs에서 값이 이전 시퀀스에서 선언된 변수를 참조하는데, validator가 이를 **전혀 검증하지 않는다.**

예시 — 아래가 validator를 통과한다:
```go
// @get Gig gig = Gig.FindByID({ID: request.GigID})
// @auth "publish" "gig" {ResourceID: blahblah.ID, UserID: nonexistent.ID} "권한 없음"
```

`blahblah`, `nonexistent`는 어디에도 선언되지 않은 변수인데 에러가 나지 않는다.

### 기대 동작

inputs의 값이 `변수.필드` 또는 `변수` 형태일 때, 해당 변수가 다음 중 하나인지 검증:

| 유효한 변수 소스 | 예시 |
|---|---|
| `request` | `request.GigID` (예약어, 항상 유효) |
| `currentUser` | `currentUser.ID` (예약어, 항상 유효) |
| 이전 시퀀스의 `@result` var | `@get Gig gig = ...` → `gig.ID` |
| 리터럴 | `"cancelled"`, 숫자 등 |

유효하지 않은 변수 참조 → **ERROR**.

### 변경 위치

#### `validator/validator.go`

시퀀스를 순회하며 스코프(선언된 변수 set)를 누적하고, 각 시퀀스의 inputs 값에서 변수명을 추출하여 스코프에 있는지 검증한다.

```go
// 의사 코드
scope := map[string]bool{"request": true, "currentUser": true}
for _, seq := range fn.Sequences {
    // inputs 값의 변수 참조 검증
    for key, val := range seq.Inputs {
        varName := extractVarName(val)  // "gig.ID" → "gig", "request.GigID" → "request"
        if varName != "" && !isLiteral(val) && !scope[varName] {
            // ERROR: 변수 varName이 선언되지 않음
        }
    }
    // 이 시퀀스의 result를 스코프에 추가
    if seq.Result != nil && seq.Result.Var != "" {
        scope[seq.Result.Var] = true
    }
}
```

### 테스트

- 유효한 참조: `request.ID`, `currentUser.ID`, 이전 `@get`의 result var → 에러 없음
- 미선언 변수 참조: `unknown.ID` → ERROR
- 리터럴: `"cancelled"` → 에러 없음

## SSaC 작성 문법 변경

없음. 문법은 변경하지 않는다.
