# 수정지시서013 실행결과

**실행 Phase**: Phase012-gin-migration
**실행일**: 2026-03-09
**결과**: ✅ 전체 통과 (60/60 테스트)

## 수정 항목별 결과

### 1. 함수 시그니처 ✅

- `func Name(w http.ResponseWriter, r *http.Request, ...)` → `func Name(c *gin.Context)`
- path param이 시그니처에서 제거됨

### 2. 경로 파라미터 추출 ✅

- 함수 본문 상단에서 `c.Param()` + 타입 변환 코드 생성
- `generatePathParamCode()` 함수 추가
- int64: `c.Param("Name")` + `strconv.ParseInt` + `c.JSON(400, gin.H{...})` 에러 처리
- string: `c.Param("Name")` 단독

### 3. 요청 본문 파싱 ✅

- JSON body (2개 이상): `json.NewDecoder(r.Body).Decode` → `c.ShouldBindJSON(&req)`
- 단일 파라미터: `r.FormValue(...)` → `c.Query(...)`
- 에러 응답: `c.JSON(http.StatusBadRequest, gin.H{...})`

### 4. 에러 응답 ✅

- 전 템플릿의 `http.Error(w, "msg", status)` → `c.JSON(status, gin.H{"error": "msg"})`
- authorize, get, guard nil/exists/state, post, put, delete, call_component, call_func 모두 변경

### 5. 성공 응답 ✅

- `w.Header().Set + json.NewEncoder(w).Encode(map...)` → `c.JSON(http.StatusOK, gin.H{...})`

### 6. authorize — currentUser 추출 ✅

- `currentUser := c.MustGet("currentUser").(*model.CurrentUser)` 코드 자동 생성
- `needsCurrentUser()` 함수로 authorize 또는 `@param ... currentUser` 사용 시 감지
- `currentUser` 템플릿 추가, 함수 본문 상단에서 한 번만 생성

### 7. import 변경 ✅

- `"github.com/gin-gonic/gin"` 항상 포함
- `"net/http"` 유지 (status code 상수)
- `"encoding/json"` 트리거 제거 (response json, json_body 모두 gin API 사용)

### 8. QueryOpts 파싱 ✅

- `generateQueryOptsCode()` 함수 추가
- `c.Query("limit")` + `strconv.Atoi` → `opts.Limit`
- `c.Query("offset")` → `opts.Offset`
- `c.Query("sort")` → `opts.SortCol`
- 심볼 테이블의 `XPagination`/`XSort` 존재 여부에 따라 조건부 생성
