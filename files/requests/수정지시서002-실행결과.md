## BUG005: sort 파라미터 SQL Injection — 수정 완료

### 변경 내용

`generateQueryOptsCode` 함수가 OpenAPI `x-sort.Allowed` 목록을 읽어 allowlist 기반 검증 코드를 생성하도록 수정.

- 시그니처 변경: `generateQueryOptsCode(st)` → `generateQueryOptsCode(funcName, st)`
- `x-sort`가 있으면 `allowedSort` map 리터럴 생성 후 `allowedSort[v]` 검증
- `direction`은 `"asc"` / `"desc"` 고정 검증
- `x-sort`가 없으면 sort/direction 코드 자체를 생성하지 않음

### 생성 코드 예시

```go
allowedSort := map[string]bool{"created_at": true, "title": true, "price": true}
if v := c.Query("sort"); allowedSort[v] {
    opts.SortCol = v
}
if v := c.Query("direction"); v == "asc" || v == "desc" {
    opts.SortDir = v
}
```

### 변경 파일

| 파일 | 변경 |
|---|---|
| `generator/go_helpers.go` | `generateQueryOptsCode` 시그니처 변경, allowlist 코드 생성 |
| `generator/go_handler.go` | 호출부 인자 추가 |
| `generator/go_handler_test.go` | `TestGenerateSortAllowlist`, `TestGenerateSortNoXSort` 추가 |

## BUG006: password_hash JSON 노출 — 범위 밖

ssac는 DDL→Go struct 변환을 하지 않는다. ssac가 생성하는 것은 model interface이며, struct 필드의 json 태그는 ssac 범위 밖이다. 이 버그는 실제 Go struct를 생성하는 측(수동 코드 또는 다른 도구)에서 수정해야 한다.

## BUG007: authz.Check에 claims 미전달 — 수정 완료

### 변경 내용

`@auth` 시퀀스에서 `currentUser.*`를 참조하면 `Claims: authz.Claims{UserID: currentUser.ID}` 및 `Role: currentUser.Role`을 자동 추가.

- `templateData`에 `ClaimsCode string` 필드 추가
- `buildTemplateData`에서 `@auth` + `hasCurrentUserRef(inputs)` 판정 시 ClaimsCode 설정 및 Role 자동 삽입
- auth/sub_auth 템플릿에 `{{.ClaimsCode}}` 삽입

### 생성 코드 예시

```go
if _, err := authz.Check(authz.CheckRequest{
    Action: "read", Resource: "project",
    Claims: authz.Claims{UserID: currentUser.ID},
    ResourceID: project.ID, Role: currentUser.Role,
}); err != nil {
    c.JSON(http.StatusForbidden, gin.H{"error": "권한이 없습니다"})
    return
}
```

`currentUser` 참조가 없는 `@auth`는 Claims/Role 없이 기존 동작 유지.

### 변경 파일

| 파일 | 변경 |
|---|---|
| `generator/go_helpers.go` | `ClaimsCode` 필드 추가, `buildTemplateData` 로직 추가 |
| `generator/go_templates.go` | auth/sub_auth 템플릿에 `{{.ClaimsCode}}` 추가 |
| `generator/go_handler_test.go` | `TestGenerateAuthClaims`, `TestGenerateAuthNoClaimsWithoutCurrentUser`, `TestGenerateSubscribeAuthClaims` 추가 |

## BUG008: @call 에러 일괄 500 반환 — 이미 구현됨

Phase026에서 구현 완료:
- 파서: `Sequence.ErrStatus int` 필드로 커스텀 HTTP 상태 코드 파싱
- 코드젠: `@call` 템플릿에서 `{{.ErrStatus}}` 사용 (기본값 500)
- 테스트: `TestGenerateCallErrStatus` 존재
