# 수정지시서 022: @auth 코드젠 변경 + @call 타입 검증 + 미사용 변수 _ 처리 + config.* 코드젠

## 현재 문제

1. SSaC `@auth`가 생성하는 코드가 fullend authzgen의 고정 `authz.Input{ID interface{}}` 구조체에 의존한다.
   SSaC `@auth` inputs에 `{UserID: ..., ResourceID: ...}` 같은 복수 필드를 전달하면 생성 코드가 컴파일되지 않는다.
2. `@call` inputs의 필드 이름만 검증하고 타입은 비교하지 않는다. DDL `INTEGER` → sqlc `int32`, func Request `int` → 타입 불일치 컴파일 에러.
3. `@call`/`@post` 결과 변수가 이후 시퀀스에서 참조되지 않으면 Go `declared and not used` 에러.
4. SSaC `config.SMTPHost` → 코드젠이 `config.SMTPHost` 그대로 출력 → `undefined: config`. config 예약 소스에 대한 코드젠 변환이 없음.

## 변경 목표

### A. @auth → @call 방식 코드 생성

`@auth`의 코드 생성을 `@call func`과 동일한 방식으로 변경한다.
- `@auth` 시맨틱: 에러 시 **403 Forbidden** 반환 (이 부분만 `@call`과 다름)
- 기본 인가 패키지: `fullend/pkg/authz` (fullend 기본 제공)
- fullend.yaml에 `authz:` 설정이 없으면 → `pkg/authz` 사용 (기본값)
- fullend.yaml에 `authz:` 설정이 있으면 → 사용자 지정 패키지 사용
- SSaC `@auth` inputs는 authz 패키지의 interface/Request struct 필드와 validate 시 비교

### B. @call func 입력 타입 검증

`@call` inputs의 필드 타입을 func Request struct 필드 타입과 비교. 불일치 시 ERROR.

### C. 미사용 변수 _ 처리

`@call`/`@post` 결과 변수가 이후 시퀀스에서 참조되지 않으면 `_`로 생성.

### D. config.* → config.Get() 코드젠 변환

SSaC에서 `config.SMTPHost` 같은 config 예약 소스를 사용하면 generator가 `config.Get("SMTP_HOST")` 호출로 변환.
- `config.Key` → `config.Get("KEY")` (PascalCase → UPPER_SNAKE_CASE 변환)
- import에 fullend `pkg/config` 자동 추가
- fullend `pkg/config` 패키지는 fullend 측에서 별도 작성 (환경변수 기반 조회)

## fullend.yaml 설정

```yaml
# 설정 안 하면 기본값: fullend/pkg/authz
authz:
  package: github.com/gigbridge/api/internal/authz   # 사용자 커스텀 패키지
```

Fallback chain:
1. `fullend.yaml`에 `authz.package` 지정 → 해당 패키지 사용
2. 미지정 → `github.com/park-jun-woo/fullend/pkg/authz` 기본 사용

## 변경 내용

### generator/generator.go — @auth 코드 생성

현재:
```go
// @auth generates:
h.Authz.Check(currentUser, "action", "resource", authz.Input{...})
```

변경 후:
```go
// @auth generates (same pattern as @call, but 403 on error):
_, err := authz.Check(authz.CheckRequest{Action: "action", Resource: "resource", UserID: currentUser.ID, ResourceID: gig.ClientID})
if err != nil {
    c.JSON(http.StatusForbidden, gin.H{"error": "Not authorized"})
    return
}
```

구체적으로:
1. `@auth` inputs의 키-값 쌍을 `@call`과 동일하게 Request struct 필드로 매핑
2. `Action`과 `Resource`는 `@auth`의 첫 두 인자에서 자동 추가
3. 에러 핸들링만 `@call`(500)과 다르게 403으로 생성
4. import 경로는 fullend.yaml `authz.package` 또는 기본값 사용

### generator/generator.go — 미사용 변수 _ 처리

1. generator가 함수 내 전체 시퀀스를 사전 스캔
2. 각 `@call`/`@post` 결과 변수가 이후 시퀀스에서 참조되는지 확인
3. 미참조 시 변수명을 `_`로 치환

```go
// 참조됨:
escrow, err := billing.HoldEscrow(...)

// 미참조:
_, err := billing.HoldEscrow(...)
```

### validator/validator.go — @call 입력 타입 검증 추가

현재 `@call` inputs에서 필드 이름 존재만 확인. 타입 비교 로직 추가.

SSaC 변수의 타입 추적:
- `@get Gig gig = Gig.FindByID(...)` → `gig`의 타입은 DDL `gigs` 테이블에서 유래
- `gig.Budget`의 타입은 DDL `budget INTEGER` → sqlc `int32`

func Request 필드 타입:
- `HoldEscrowRequest.Amount int`

비교:
- `gig.Budget`(int32) ≠ `Amount`(int) → ERROR: `@call billing.HoldEscrow 입력 "Amount"의 타입 불일치: gig.Budget은 int32, Request 필드는 int`

### generator/generator.go — config.* 코드젠 변환

현재:
```go
// SSaC: config.SMTPHost
// 생성 코드:
config.SMTPHost   // → undefined: config 에러
```

변경 후:
```go
// SSaC: config.SMTPHost
// 생성 코드:
config.Get("SMTP_HOST")
```

구체적으로:
1. generator가 시퀀스 내 `config.XXX` 패턴을 인식
2. `config.PascalKey` → `config.Get("UPPER_SNAKE_KEY")` 변환
   - `SMTPHost` → `SMTP_HOST`
   - `DatabaseURL` → `DATABASE_URL`
   - `AppPort` → `APP_PORT`
3. config 패턴이 하나라도 존재하면 import에 fullend `pkg/config` 경로 자동 추가
4. PascalCase → UPPER_SNAKE_CASE 변환 규칙: 대문자 앞에 `_` 삽입 후 전체 대문자화 (단, 연속 대문자 그룹은 하나로 유지: `SMTP` → `SMTP`)

### parser/parser.go — 변경 없음

`@auth` 파서는 현재 그대로 유지. 이미 `action`, `resource`, `inputs`, `message`를 올바르게 파싱하고 있음.
`config.*` 도 파서 변경 불필요 — 기존에 `config.Key` 를 변수 참조로 올바르게 파싱하고 있음. 변환은 generator 단계에서 처리.

## SSaC 사용법 (변경 전후 동일)

```go
// @auth "AcceptProposal" "gig" {UserID: currentUser.ID, ResourceID: gig.ClientID} "Not authorized"
```

import 불필요 — generator가 fullend.yaml 설정 또는 기본값에서 authz 패키지 경로를 자동 결정.

## fullend/pkg/authz 기본 제공 패키지 (fullend 측 별도 작성)

```go
package authz

type CheckRequest struct {
    Action     string
    Resource   string
    UserID     int64
    ResourceID int64
}

type CheckResponse struct{}

func Check(req CheckRequest) (CheckResponse, error) {
    // 기본 구현: OPA Rego 기반
    return CheckResponse{}, nil
}
```

- SSaC `@auth` inputs는 이 interface의 필드와 validate 시 비교
- 사용자가 커스텀 패키지를 지정하면 해당 패키지의 CheckRequest 필드와 비교

## SSaC config 사용법

```go
// SSaC에서:
// @call mail.SendTemplateEmail {Host: config.SMTPHost, Port: config.SMTPPort, From: config.SMTPFrom, ...}

// 생성 코드:
_, err := mail.SendTemplateEmail(mail.SendTemplateEmailRequest{
    Host: config.Get("SMTP_HOST"),
    Port: config.Get("SMTP_PORT"),
    From: config.Get("SMTP_FROM"),
    ...
})
```

## fullend/pkg/config 기본 제공 패키지 (fullend 측 별도 작성)

```go
package config

import "os"

// Get returns the environment variable value for the given key.
// Returns empty string if not set.
func Get(key string) string {
    return os.Getenv(key)
}
```

## 영향 범위

- `generator/generator.go`: `@auth` 코드 생성 로직 + `@call`/`@post` 미사용 변수 `_` 처리 + `config.*` → `config.Get()` 변환
- `validator/validator.go`: `@call` 입력 타입 비교 검증 추가
- 테스트: `generator/generator_test.go`에 `@auth` → `@call` 스타일 출력 테스트 + 미사용 변수 `_` 테스트 + `config.*` 변환 테스트
- 테스트: `validator/validator_test.go`에 `@call` 타입 불일치 ERROR 테스트

## fullend 측 연동 변경 (별도)

- `pkg/authz/` 신규 작성 (기본 OPA Rego 기반 구현)
- `pkg/config/` 신규 작성 (환경변수 기반 조회 함수)
- `fullend.yaml` 파서: `authz.package` 필드 추가
- `authzgen`: 자동 생성 제거 → `pkg/authz` 또는 사용자 패키지를 `@call func`과 동일하게 복사
- crosscheck: `@auth` inputs → authz 패키지 CheckRequest 필드 매칭 검증
