# 수정지시서 014: pagination 제네릭 타입 + @response 간단쓰기 + parseInputs 검증

## 1. parseInputs에서 콜론 없는 입력 → ERROR

### 현상

`Gig.List({query})`처럼 콜론 없는 입력을 파서가 조용히 무시한다.

### 원인

`parser.go`의 `parseInputs()`:

```go
colonIdx := strings.IndexByte(pair, ':')
if colonIdx < 0 {
    continue  // ← {query} 같은 콜론 없는 입력이 무시됨
}
```

### 수정

콜론 없는 비어있지 않은 항목이 있으면 파싱 에러 처리:

```
ERROR: list_gigs.go:ListGigs — "{query}"는 유효하지 않은 입력 형식입니다. "{Key: value}" 형식을 사용하세요.
```

## 2. Page[T], Cursor[T] 제네릭 타입 지원

### 배경

pagination 응답을 엔티티별 DTO 대신 제네릭 래퍼로 처리한다. fullend `pkg/pagination/`에 타입 정의가 제공된다.

```go
// fullend/pkg/pagination/page.go
type Page[T any] struct {
    Items []T   `json:"items"`
    Total int64 `json:"total"`
}

// fullend/pkg/pagination/cursor.go
type Cursor[T any] struct {
    Items      []T    `json:"items"`
    NextCursor string `json:"next_cursor"`
    HasNext    bool   `json:"has_next"`
}
```

### SSaC parser 변경

`@get`의 반환 타입에서 `Page[Gig]`, `Cursor[Gig]` 형식을 파싱할 수 있어야 한다:

```go
// @get Page[Gig] gigPage = Gig.List({Query: query})
```

- 타입 파싱: `Page[Gig]` → 제네릭 래퍼 `Page`, 내부 타입 `Gig`
- `Cursor[Gig]`도 동일

### SSaC validator 변경

OpenAPI `x-pagination`과 교차 검증:

```
✗ x-pagination: style: offset 인데 @get 반환이 Page[T]가 아님 → ERROR
✗ x-pagination: style: cursor 인데 @get 반환이 Cursor[T]가 아님 → ERROR
✗ x-pagination 없는데 @get 반환이 []T가 아님 → ERROR (Page[T], Cursor[T] 사용 불가는 아님)
```

수정: x-pagination 없을 때 Page[T]/Cursor[T] 사용은 허용. 단, x-pagination 있을 때 타입 불일치만 ERROR.

```
✗ x-pagination: style: offset 인데 @get 반환이 Page[T]가 아님 → ERROR
✗ x-pagination: style: cursor 인데 @get 반환이 Cursor[T]가 아님 → ERROR
✓ x-pagination 없는데 Page[T] 사용 → OK (일관성 위해 허용)
✗ x-pagination: style: offset 인데 @response에 gigPage.Total 없음 → ERROR (풀어쓰기 시)
```

### SSaC generator 변경

`Page[T]`, `Cursor[T]` 반환 시 model interface 생성:

```go
// Page[Gig] → model interface
List(opts QueryOpts) (*pagination.Page[Gig], error)

// Cursor[Gig] → model interface
List(opts QueryOpts) (*pagination.Cursor[Gig], error)
```

생성된 handler 코드에 `import "github.com/park-jun-woo/fullend/pkg/pagination"` 추가.

## 3. @response 간단쓰기 지원

### 현재

`@response`는 항상 `{ }` 블록으로 필드 매핑 필요:

```go
// @response {
//   gigs: gigs
//   total: total
// }
```

### 변경

`@response 변수명` 형식 추가 — struct를 그대로 JSON 직렬화:

```go
// 간단쓰기 — struct 그대로 반환
// @get Page[Gig] gigPage = Gig.List({Query: query})
// @response gigPage
// → c.JSON(200, gigPage)

// 풀어쓰기 — 필드 매핑 (기존 방식, 계속 지원)
// @get Gig gig = Gig.FindByID({ID: id})
// @response {
//   gig: gig
// }
// → c.JSON(200, gin.H{"gig": gig})
```

parser: `@response` 다음에 `{`가 오면 기존 필드 매핑, 변수명이 오면 직접 반환.

## 검증 방법

```go
// 1. parseInputs 에러
// @get []Gig gigs = Gig.List({query})  → ERROR

// 2. Page[T] 파싱
// @get Page[Gig] gigPage = Gig.List({Query: query})  → OK

// 3. @response 간단쓰기
// @response gigPage  → c.JSON(200, gigPage)

// 4. x-pagination 교차 검증
// x-pagination: style: offset + @get []Gig  → ERROR
// x-pagination: style: offset + @get Page[Gig]  → OK
```
