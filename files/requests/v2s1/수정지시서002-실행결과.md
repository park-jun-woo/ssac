# 수정지시서 002 실행결과: @call 코드젠 수정 + validateCurrentUserType 제거

## 실행일: 2026-03-10

## 문제 1: guard-style @call `_, err` 패턴

### 변경: generator/go_templates.go

`call_no_result` 템플릿에 `_,` 추가:

```
// 변경 전
if err {{if .FirstErr}}:={{else}}={{end}} pkg.Func(pkg.FuncRequest{...}); err != nil {

// 변경 후
if _, err {{if .FirstErr}}:={{else}}={{end}} pkg.Func(pkg.FuncRequest{...}); err != nil {
```

### 생성 코드 확인

```go
if _, err = auth.VerifyPassword(auth.VerifyPasswordRequest{user.PasswordHash, password}); err != nil {
```

## 문제 2: @call 위치 기반 필드명 매핑

### 변경: generator/go_target.go

`buildCallInputFields`를 위치 기반 초기화(unkeyed literal)로 변경. SSaC는 외부 패키지의 Request 구조체 필드 정보를 가지고 있지 않으므로, 필드명 대신 값만 나열한다.

```go
// 변경 전 (named fields — 소스 필드명을 구조체 필드명으로 오용)
auth.IssueTokenRequest{ID: user.ID, Email: user.Email, Role: user.Role}

// 변경 후 (unkeyed — 위치 기반)
auth.IssueTokenRequest{user.ID, user.Email, user.Role}
```

### 생성 코드 확인

```go
billing.CalculateRefund(billing.CalculateRefundRequest{reservation.ID, reservation.StartAt, reservation.EndAt})
```

## 문제 3: validateCurrentUserType 제거

### 변경 파일

| 파일 | 내용 |
|---|---|
| `validator/validator.go` | `validateCurrentUserType` 함수 삭제, `ValidateWithSymbols`에서 호출 제거 |
| `validator/symbol.go` | `HasCurrentUserType bool` 필드 삭제, `loadGoInterfaces`에서 CurrentUser 감지 로직 삭제 |
| `validator/validator_test.go` | `TestValidateCurrentUserTypeMissing`, `TestValidateCurrentUserTypeExists` 삭제 |

### 근거

CurrentUser는 인증 인프라(fullend.yaml claims 설정)가 담당하며, SSaC가 `model/`에 CurrentUser 타입 존재를 요구하는 것은 월권이다. SSaC는 `currentUser`가 예약 소스라는 것만 알면 된다.
