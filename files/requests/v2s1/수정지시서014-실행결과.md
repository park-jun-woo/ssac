## 변경 내용

### 1. parseInputs 에러 처리

**parser/parser.go**

`parseInputs()` 반환 타입을 `(map[string]string, error)`로 변경. 콜론 없는 입력(`{query}`)에 에러 반환:

```go
if colonIdx < 0 {
    return nil, fmt.Errorf("%q는 유효하지 않은 입력 형식입니다. \"{Key: value}\" 형식을 사용하세요", pair)
}
```

`extractInputs()`, `parseCallExprInputs()` → 에러 전파. `parseCRUD()`, `parseState()`, `parseAuth()`, `parseCall()` → `(*Sequence, error)` 반환. `parseLine()` → `(*Sequence, bool, error)` 반환. `parseComments()` → `([]Sequence, error)` 반환. `ParseFile()` → 에러 전파.

### 2. Page[T] / Cursor[T] 타입 파싱

**parser/types.go**

`Result` 구조체에 `Wrapper` 필드 추가:

```go
type Result struct {
    Type    string // "Gig" (내부 타입)
    Var     string // "gigPage"
    Wrapper string // "Page", "Cursor", ""
}
```

**parser/parser.go**

`parseResult()`에서 `Page[Gig]` 형식 인식:

```go
if bracketIdx := strings.IndexByte(typeName, '['); bracketIdx > 0 {
    r.Wrapper = typeName[:bracketIdx]   // "Page"
    r.Type = typeName[bracketIdx+1 : len(typeName)-1]  // "Gig"
}
```

### 3. @response 간단쓰기

**parser/parser.go**

`parseLine()`에서 `@response` 다음 토큰이 변수명이면 직접 반환:

```go
// @response gigPage → Sequence{Type: SeqResponse, Target: "gigPage"}
```

### 4. validator — x-pagination 교차검증 + @response 간단쓰기 필드 검증

**validator/validator.go**

`validatePaginationType()` 추가:
- `x-pagination: style: offset` + `Wrapper != "Page"` → ERROR
- `x-pagination: style: cursor` + `Wrapper != "Cursor"` → ERROR
- `x-pagination` 없음 + `Wrapper != ""` → ERROR

`validateResponse()` — 간단쓰기 시 Wrapper 고정 필드로 OpenAPI 대조:

```go
var wrapperFieldsMap = map[string][]string{
    "Page":   {"items", "total"},
    "Cursor": {"items", "next_cursor", "has_next"},
}
```

- `@response gigPage` (Page[T]) → OpenAPI에 `items`, `total` 필드 있는지 검증
- `@response gigCursor` (Cursor[T]) → OpenAPI에 `items`, `next_cursor`, `has_next` 필드 있는지 검증
- 역방향도 동작: OpenAPI에 있는 필드가 Wrapper 고정 필드에 없으면 ERROR

### 5. generator — 코드젠

**generator/go_target.go**

- `templateName()`: `@response`에 `Target` 있으면 `"response_direct"` 템플릿 사용
- `HasTotal` 판단: `Wrapper != ""`이면 `false` (Page struct가 total 포함)
- `deriveReturnType()`: `Wrapper` 있으면 `(*pagination.Page[Gig], error)` 반환
- `collectImports()`: `Wrapper` 사용 시 `pagination` import 추가
- `renderInterfaces()`: pagination import 처리
- `generateQueryOptsCode()`: nil `st` 방어

**generator/go_templates.go**

`response_direct` 템플릿 추가:

```go
{{- define "response_direct" -}}
    c.JSON(http.StatusOK, {{.Target}})
{{end}}
```
