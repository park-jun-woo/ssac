## 변경 내용

### 1. 파서 — 패키지 접두사 파싱

**parser/types.go**

`Sequence`에 `Package` 필드 추가:

```go
type Sequence struct {
    Package string // "session" (패키지 접두사, 없으면 "")
    Model   string // "Session.Get" (기존 Model.Method 형식 유지)
    // ...
}
```

**parser/parser.go**

`splitPackagePrefix()` 함수 추가. `parseCRUD()`에서 3-part 모델명 분리:

```go
func splitPackagePrefix(model string) (pkg, rest string) {
    // "session.Session.Get" → ("session", "Session.Get")
    // "User.FindByID" → ("", "User.FindByID")
    // 첫 번째 part가 소문자로 시작하면 패키지 접두사로 판단
}
```

### 2. 검증 — 패키지 모델 분기

**validator/validator.go**

`validateModel()`에서 `seq.Package != ""`일 때:
- `st.Models["pkg.Model"]` 키로 조회
- interface 없으면 WARNING (검증 불가)
- 메서드 없으면 ERROR + 사용 가능 메서드 목록 안내

```go
if seq.Package != "" {
    pkgModelKey := seq.Package + "." + modelName  // "session.Session"
    ms, ok := st.Models[pkgModelKey]
    if !ok {
        // WARNING: 검증 불가
    }
    if !ms.HasMethod(methodName) {
        // ERROR: 메서드 "Gett" 없음. 사용 가능: Delete, Get, Set
    }
}
```

### 3. 심볼 테이블 — 패키지 interface 로딩

**validator/symbol.go**

`LoadPackageInterfaces()` 추가:
1. 서비스 파일에서 패키지 접두사 모델 수집
2. import 경로에서 패키지명 매핑
3. 해당 경로의 Go interface 파싱 → `st.Models["pkg.Model"]` 등록
4. interface 이름에서 `Model` suffix 자동 제거 (`SessionModel` → `Session`)

### 4. 코드젠

**generator/go_target.go**

- `buildTemplateData()`: 패키지 모델이든 일반 모델이든 `modelName + "Model." + method` 동일 형태
- `collectModelUsages()`: `seq.Package != ""` 모델은 `models_gen.go`에 포함하지 않음
- `defaultMessage()`: 패키지 접두사 포함 메시지 (`"session.Session 조회 실패"`)
