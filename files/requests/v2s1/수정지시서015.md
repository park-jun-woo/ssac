# 수정지시서 015: @response 간단쓰기 시 handler에 불필요한 pagination import 제거

## 현상

`@response gigPage` 간단쓰기를 사용하면 handler에 `pagination` import가 추가되지만, handler 코드에서 pagination 패키지를 직접 참조하지 않아 `imported and not used` 컴파일 에러 발생.

```go
// 생성된 handler
import (
    "github.com/park-jun-woo/fullend/pkg/pagination" // ← 사용 안 됨
)

func (h *Handler) ListGigs(c *gin.Context) {
    gigPage, err := h.GigModel.List(opts)
    c.JSON(http.StatusOK, gigPage) // pagination 타입 직접 참조 없음
}
```

## 원인

`collectImports()`에서 `Wrapper != ""`이면 무조건 pagination import 추가. 하지만 `@response gigPage` 간단쓰기에서는 handler가 `pagination.Page[T]`를 직접 참조하지 않음. model interface가 `*pagination.Page[T]`를 반환하므로 model 패키지에서만 import가 필요.

## 수정

handler의 `collectImports()`에서 pagination import 조건:
- `@response { }` 풀어쓰기로 `gigPage.Items`, `gigPage.Total` 등 필드 접근 시 → import 필요 (타입 단언 등)
- `@response gigPage` 간단쓰기 시 → import 불필요

실제로 간단쓰기에서는 `c.JSON(200, gigPage)`만 하므로 handler에서 pagination 타입을 알 필요 없음. **간단쓰기일 때 handler import에서 pagination을 제외**.

## 검증 방법

```go
// @get Page[Gig] gigPage = Gig.List({Query: query})
// @response gigPage
```

생성된 handler에 `"github.com/park-jun-woo/fullend/pkg/pagination"` import 없음 확인.

`go build ./...` 통과.
