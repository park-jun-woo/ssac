# 수정지시서 028 실행결과

## Part 1: 파서 — ErrStatus 파싱 확장 ✅

### 변경 내용

#### `parser/types.go`

`ErrStatus` 주석을 `@call` 전용에서 전체 타입별 기본값 설명으로 수정:

```go
ErrStatus    int    // 에러 HTTP 상태 코드 (0이면 타입별 기본값: @call→500, @empty→404, @exists→409, @state→409, @auth→403)
```

#### `parser/parser.go`

**`splitTargetMessage`** — 2-tuple → 3-tuple 반환. remainder로 trailing 숫자를 전달.

**`parseTwoQuoted`** — 2-tuple → 3-tuple 반환. remainder로 trailing 숫자를 전달.

**`parseGuard`** (@empty/@exists) — remainder가 100-599 범위 숫자이면 `seq.ErrStatus`에 저장:

```go
target, msg, remainder := splitTargetMessage(rest)
// remainder → strconv.Atoi → ErrStatus
```

**`parseState`** — `parseTwoQuoted` 3번째 반환값에서 ErrStatus 파싱.

**`parseAuth`** — `extractQuoted` 결과의 remainder에서 ErrStatus 파싱.

#### `parser/parser_test.go` — 4개 추가

- `TestParseEmptyErrStatus`: `@empty target "msg" 402` → ErrStatus=402
- `TestParseExistsErrStatus`: `@exists target "msg" 422` → ErrStatus=422
- `TestParseStateErrStatus`: `@state ... "msg" 422` → ErrStatus=422
- `TestParseAuthErrStatus`: `@auth ... "msg" 401` → ErrStatus=401

## Part 2: 코드젠 — ErrStatus 반영 ✅

### 변경 내용

#### `generator/go_target.go`

`buildTemplateData()`에 empty/exists/state/auth ErrStatus 처리 switch 추가. ErrStatus가 0이면 타입별 기본값, 아니면 `httpStatusConst()`로 변환.

`httpStatusConst()`에 422 → `http.StatusUnprocessableEntity` 매핑 추가.

#### `generator/go_templates.go`

4개 템플릿의 하드코딩 HTTP 상태 코드를 `{{.ErrStatus}}`로 교체:

| 템플릿 | before | after |
|---|---|---|
| `empty` | `http.StatusNotFound` | `{{.ErrStatus}}` |
| `exists` | `http.StatusConflict` | `{{.ErrStatus}}` |
| `state` | `http.StatusConflict` | `{{.ErrStatus}}` |
| `auth` | `http.StatusForbidden` | `{{.ErrStatus}}` |

subscribe 템플릿(`sub_empty`, `sub_exists`, `sub_state`, `sub_auth`)은 HTTP 상태 코드를 사용하지 않으므로 변경 없음.

#### `generator/generator_test.go` — 4개 추가

- `TestGenerateEmptyErrStatus`: 402 → `http.StatusPaymentRequired` 확인
- `TestGenerateExistsErrStatus`: 422 → `http.StatusUnprocessableEntity` 확인
- `TestGenerateStateErrStatus`: 422 → `http.StatusUnprocessableEntity` 확인
- `TestGenerateAuthErrStatus`: 401 → `http.StatusUnauthorized` 확인
