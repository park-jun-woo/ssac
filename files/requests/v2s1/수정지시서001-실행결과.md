# 수정지시서 001 실행결과: @get 0-arg 허용

## 실행일: 2026-03-10

## 수정지시서 원안 대비 변경

원안은 `@get`/`@post`/`@put`/`@delete` 모두 0-arg 허용이었으나, 검토 결과 타입별 차등 적용:

| 타입 | Args 0개 | 근거 |
|---|---|---|
| `@get` | 허용 | 전체 조회 (`Course.List()`) |
| `@post` | ERROR | 입력 없이 생성 불가 |
| `@put` | ERROR | 입력 없이 수정 불가 |
| `@delete` | WARNING | 전체 삭제 가능하지만 위험, 의도 확인 필요 |

## 변경 내용

### validator/validator.go

`validateRequiredFields`에서 `@get`/`@post`/`@put`/`@delete`를 개별 case로 분리:

- `@get`: Args 검증 제거 (0개 허용)
- `@post`: Args 필수 유지 (ERROR)
- `@put`: Args 필수 유지 (ERROR)
- `@delete`: Args 0개 시 WARNING ("전체 삭제 의도가 맞는지 확인하세요")

### CLAUDE.md

타입별 필수 요소 테이블 업데이트:

| 타입 | 변경 전 | 변경 후 |
|---|---|---|
| get, post | Model, Result, Args | get: Model, Result / post: Model, Result, Args |
| put, delete | Model, Args | Model, Args (delete 0-arg는 WARNING) |

# Phase 007 실행결과: WARNING 억제 (`!` 접미사)

## 실행일: 2026-03-10

## 개요

모든 시퀀스 타입에 `!` 접미사를 붙이면 해당 시퀀스의 WARNING을 억제한다. ERROR는 영향 없음.

```go
// @delete! Room.DeleteAll()              — 0-arg WARNING 억제
// @response! { room: room }              — stale 데이터 WARNING 억제
```

## 변경 내용

### parser/types.go

`Sequence` 구조체에 `SuppressWarn bool` 필드 추가.

### parser/parser.go

- `parseLine`에서 `@type!` 감지: `!`가 첫 번째 공백 앞에 있으면 제거 후 `SuppressWarn = true`
- `@response!` 멀티라인 블록: `responseSuppressWarn` 변수로 전달
- if-chain → switch-case 리팩토링

### parser/parser_test.go

4개 테스트 추가:
- `TestParseSuppressWarnDelete` — `@delete!` → SuppressWarn=true
- `TestParseSuppressWarnGet` — `@get!` → SuppressWarn=true
- `TestParseNoSuppressWarn` — `@get` (! 없음) → SuppressWarn=false
- `TestParseSuppressWarnResponse` — `@response!` → SuppressWarn=true

### validator/validator.go

WARNING 발행 전 `seq.SuppressWarn` 체크:
- `@delete` 0-arg WARNING: `!seq.SuppressWarn` 조건 추가
- stale response WARNING: `seq.SuppressWarn`이면 continue

### validator/validator_test.go

2개 테스트 추가:
- `TestValidateDeleteNoArgsSuppressed` — `@delete!` 0-arg, WARNING 없음 확인
- `TestValidateStaleResponseSuppressed` — `@response!` stale WARNING 없음 확인

### CLAUDE.md

DSL 문법 섹션에 `@type!` 설명 추가.

## 코드젠 영향 검증

dummy-study를 `/tmp/ssac-phase7-check/`로 코드젠하여 7개 서비스 파일 + model/models_gen.go 전수 점검:

| 점검 항목 | 결과 |
|---|---|
| `!` 문자 누출 (함수명, 변수명, 타입명) | 없음 |
| `SuppressWarn` IR 필드 누출 | 없음 |
| `request.` 미변환 | 없음 |
| Go naming 위반 (`iD`, `Id`) | 없음 |
| 문법 오류 | 없음 |

`!`는 파서에서 제거되어 `Sequence.Type`에 `"delete"` (not `"delete!"`)로 저장되므로 코드젠 템플릿에 영향 없음.
