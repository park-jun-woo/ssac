## 변경 내용

### 1. parser/types.go

`ServiceFunc`에 `Subscribe *SubscribeInfo` 필드 추가:

```go
type ServiceFunc struct {
    Name      string
    FileName  string
    Domain    string
    Sequences []Sequence
    Imports   []string
    Subscribe *SubscribeInfo // nil이면 HTTP 트리거
}

type SubscribeInfo struct {
    Topic string // "order.completed"
}
```

`Sequence`에 `Topic`, `Options` 필드 추가:

```go
Topic   string            // "order.completed"
Options map[string]string // {delay: "1800"} (선택)
```

상수 및 유효 타입 추가:

```go
SeqPublish = "publish"
ValidSequenceTypes에 SeqPublish: true 추가
```

### 2. parser/parser.go

**@subscribe 파싱**:

`parseLine()`에서 `@subscribe "topic"` 감지 시 `Type="subscribe"` 특수 Sequence 반환. `ParseFile()`에서 추출 후 `ServiceFunc.Subscribe`에 설정하고 `Sequences`에서 필터링:

```go
var filtered []Sequence
for _, seq := range sequences {
    if seq.Type == "subscribe" {
        sf.Subscribe = &SubscribeInfo{Topic: seq.Topic}
        continue
    }
    filtered = append(filtered, seq)
}
sf.Sequences = filtered
```

**@publish 파싱**:

`parsePublish()` 함수 추가. `extractQuoted()`로 topic, `extractInputs()`로 payload와 options 추출:

```go
func parsePublish(rest string) (*Sequence, error) {
    topic, rest := extractQuoted(rest)
    payload, rest, err := extractInputs(rest)
    var options map[string]string
    if strings.HasPrefix(rest, "{") {
        options, _, err = extractInputs(rest)
    }
    return &Sequence{Type: SeqPublish, Topic: topic, Inputs: payload, Options: options}, nil
}
```

### 3. validator/validator.go

**@publish 필수 필드 검증**:

`validateRequiredFields()`에 `SeqPublish` case 추가 — Topic, Inputs(payload) 누락 검사.

**subscribe 규칙 검증**:

`validateSubscribeRules()` 함수 추가:
- `sf.Subscribe != nil` + `@response` → ERROR
- `sf.Subscribe != nil` + `request.*` 사용 → ERROR
- `sf.Subscribe == nil` + `message.*` 사용 → ERROR

**message 예약 변수**:

- `validateVariableFlow()`에서 `sf.Subscribe != nil`일 때 `declared["message"] = true`
- `reservedSources`에 `"message": true` 추가

### 4. generator/go_target.go

**templateData 확장**: `Topic`, `OptionCode` 필드 추가.

**buildTemplateData()**: `@publish` 처리 — `buildPublishPayload()`/`buildPublishOptions()` 호출.

**buildPublishPayload()**: `map[string]string` → `"Key": value` 형식 (ucFirst 키, inputValueToCode 값 변환).

**buildPublishOptions()**: delay → `queue.WithDelay(N)`, priority → `queue.WithPriority("level")`.

**collectImports()**: `@publish` 시퀀스 있으면 `"queue"` import 추가.

**err 추적**: `SeqPublish`를 `SeqPut, SeqDelete`와 동일하게 처리.

**templateName()**: `SeqPublish` → `"publish"`.

## 주요 설계 결정

1. **ctx 문제**: gin 핸들러의 `c *gin.Context`에서 `c.Request.Context()`로 표준 `context.Context` 전달

3. **err 추적**: `@publish`는 `if err :=` 인라인이 아닌 `err =/err :=` + `if err != nil` 패턴 (다른 시퀀스와 일관성)
4. **queue import**: `collectImports()`에서 `@publish` 시퀀스 감지 시 자동 추가
