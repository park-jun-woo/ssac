# 수정지시서 027 실행결과

## Part 1: @auth 코드젠 — Role 자동 추가 ✅

### 변경 내용

#### `generator/go_target.go`

`buildSequenceData()` 내 `@auth` InputFields 생성 분기에서, Inputs에 `currentUser.*` 참조가 있고 `Role` 키가 없으면 `Role: currentUser.Role`을 자동 삽입하도록 변경.

- `hasCurrentUserRef()`: inputs 값 중 `currentUser.` 접두사 존재 여부 확인
- `copyInputs()`: 원본 seq.Inputs를 수정하지 않도록 얕은 복사

**생성 결과 (before)**:
```go
authz.Check(authz.CheckRequest{Action: "PublishGig", Resource: "gig", ResourceID: gig.ID, UserID: currentUser.ID})
```

**생성 결과 (after)**:
```go
authz.Check(authz.CheckRequest{Action: "PublishGig", Resource: "gig", ResourceID: gig.ID, Role: currentUser.Role, UserID: currentUser.ID})
```

- `currentUser.*` 참조 없는 @auth → Role 미생성
- SSaC에 명시적 `Role` 키가 있으면 자동 추가 안 함 (덮어쓰기 방지)

#### `generator/generator_test.go`

- `TestGenerateAuthCallStyle`: `Role: currentUser.Role` 포함 확인 추가
- `TestGenerateAuthNoCurrentUser`: `Role:` 미포함 확인 추가

## Part 3: validator 자체 점검 — 추가 보완 ✅

```

```

