# 수정지시서 004 실행결과: @auth 코드젠 — authz.Claims 제거 + CheckRequest 필드 정리

## 실행일: 2026-03-13

## 변경 내용

### 1. `ClaimsCode` 필드 제거

`generator/go_helpers.go`의 `templateData` 구조체에서 `ClaimsCode string` 필드 삭제.

### 2. `UserID` 직접 주입

`@auth` + `currentUser` 참조 시 `ClaimsCode` 대신 inputs에 `UserID: currentUser.ID` 직접 추가:

```go
// 변경 전
d.ClaimsCode = "Claims: authz.Claims{UserID: currentUser.ID}, "

// 변경 후
if _, hasUserID := inputs["UserID"]; !hasUserID {
    inputs["UserID"] = "currentUser.ID"
}
```

이미 `UserID`가 inputs에 명시되어 있으면 중복 추가하지 않음.

### 3. 템플릿 `{{.ClaimsCode}}` 제거

`generator/go_templates.go`의 `auth`, `sub_auth` 템플릿에서 `{{.ClaimsCode}}` 제거.

### 생성 코드 변경

```go
// 변경 전 (컴파일 에러)
authz.Check(authz.CheckRequest{
    Action: "PublishGig",
    Resource: "gig",
    Claims: authz.Claims{UserID: currentUser.ID},  // 미존재 타입
    ResourceID: gig.ID,
    Role: currentUser.Role,
    UserID: currentUser.ID,                          // 중복
})

// 변경 후
authz.Check(authz.CheckRequest{
    Action: "PublishGig",
    Resource: "gig",
    ResourceID: gig.ID,
    Role: currentUser.Role,
    UserID: currentUser.ID,
})
```

## 변경 파일

| 파일 | 변경 |

| `generator/go_helpers.go` | `ClaimsCode` 필드 제거, `UserID` inputs 직접 추가 |

| `generator/go_handler_test.go` | `Claims: authz.Claims{...}` assertion → `UserID: currentUser.ID` |
