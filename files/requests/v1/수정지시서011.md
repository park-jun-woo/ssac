# 수정지시서 011: `@sequence password` 폐지 + call 패키지 분리 코드젠

## 요청자
fullend

## 요약

두 가지 변경:

1. **`@sequence password` 폐지** — bcrypt 하드코딩 제거, `@sequence call`로 통합. 시퀀스 타입 11→10.
2. **`@func` 패키지 분리 코드젠** — `@func hashPassword` → `@func auth.hashPassword`. SSaC는 외부 패키지의 `func(In) (Out, error)` 메서드를 **호출하는 코드만 생성**한다. Input/Output struct 및 func 본체는 fullend가 별도 관리.

## 1. call 호출 형태 변경

### 수정 전
```go
// @sequence call
// @func hashPassword
// @param Password request
// @result HashedPassword string
```
→ Handler에 `HashPassword func(args ...interface{}) (interface{}, error)` 필드
→ 생성: `h.HashPassword(password)`

### 수정 후
```go
// @sequence call
// @func auth.hashPassword
// @param Password request
// @result HashedPassword string
```
→ Handler에 `Auth *auth.Service` 필드
→ 생성: `h.Auth.HashPassword(auth.HashPasswordInput{Password: password})`

## 2. SSaC의 책임 범위

SSaC가 하는 것:
- `@func auth.hashPassword` 파싱 (Package="auth", Func="hashPassword")
- Handler struct에 `Auth *auth.Service` 필드 생성
- 서비스 코드에서 `h.Auth.HashPassword(auth.HashPasswordInput{...})` 호출 생성
- import에 `"<module>/internal/auth"` 추가

SSaC가 하지 않는 것:
- Input/Output struct 정의 (fullend가 func 스펙 파일에서 생성)
- func 본체 구현 (LLM이 구현)
- auth 패키지 파일 생성 (fullend gen이 관리)

## 3. `@func` 파싱 규칙

```
@func <package>.<funcName>
```
- `package`: 소문자 (`auth`, `payment`, `notification`)
- `funcName`: camelCase (`hashPassword`, `verifyPassword`)
- `.`이 없으면 validator ERROR

### Sequence struct 변경

```go
// parser/types.go
type Sequence struct {
    Type      string
    // ... 기존 필드
    Func      string  // 기존: "hashPassword" → 유지 (funcName만)
    Package   string  // 신규: "auth" (패키지명)
}
```

## 4. 코드젠 변경

### Handler struct

```go
// 수정 전
type Handler struct {
    UserModel    model.UserModel
    HashPassword func(args ...interface{}) (interface{}, error)
    IssueToken   func(args ...interface{}) (interface{}, error)
}

// 수정 후
type Handler struct {
    UserModel model.UserModel
    Auth      *auth.Service      // @func auth.* → 패키지당 1개 필드
}
```

같은 패키지의 func이 여러 개여도 필드는 하나:
```go
// @func auth.hashPassword   ─┐
// @func auth.verifyPassword  ├→ Auth *auth.Service (1개)
// @func auth.issueToken     ─┘
```

### 서비스 코드 생성

```go
// value형 (@result 있음)
out, err := h.Auth.HashPassword(auth.HashPasswordInput{Password: password})
if err != nil {
    http.Error(w, "hashPassword 호출 실패", http.StatusInternalServerError)
    return
}
hashedPassword := out.HashedPassword

// guard형 (@result 없음, @message 있음)
_, err = h.Auth.VerifyPassword(auth.VerifyPasswordInput{
    PasswordHash: user.PasswordHash,
    Password:     password,
})
if err != nil {
    http.Error(w, "비밀번호가 일치하지 않습니다", http.StatusUnauthorized)
    return
}
```

### Input struct 필드 매핑 규칙

`@param` → Input struct 필드명:
- `@param Password request` → `Password: password` (request에서 추출한 변수)
- `@param user.PasswordHash` → `PasswordHash: user.PasswordHash` (이전 시퀀스 결과 참조)

### import 처리

SSaC는 import를 자체 생성하지 않는다. **SSaC 스펙 파일(Go 소스)에 이미 선언된 import를 그대로 사용한다.**

SSaC 파서가 Go AST로 파일의 import를 읽고, `@func auth.hashPassword`의 `auth`가 어떤 패키지 경로인지 import 선언에서 확인한다. 코드젠 시 해당 import를 생성 코드에 그대로 옮긴다.

```go
// SSaC 스펙 파일 (specs/dummy-study/service/login.go)
package service

import (
    "net/http"
    auth "github.com/park-jun-woo/fullend/pkg/auth"  // ← 여기서 경로 결정
)

// @sequence call
// @func auth.verifyPassword
// @param user.PasswordHash
// @param Password request
// @message "비밀번호가 일치하지 않습니다" 401
func Login(w http.ResponseWriter, r *http.Request) {}
```

커스텀 구현을 쓰고 싶으면 스펙 파일에서 import 경로만 바꾸면 된다:
```go
import auth "dummy-study/backend/internal/auth"  // 커스텀
```

## 6. 변경 파일

### parser/types.go
- `SeqPassword` 상수 및 `ValidSequenceTypes` 항목 제거 (11→10)
- `Sequence` struct에 `Package string` 필드 추가

### parser/parser.go
- `@func` 파싱: `.`으로 split → `seq.Package`, `seq.Func` 설정
- `password` 관련 파싱 분기 제거

### validator/validator.go
- `case parser.SeqPassword:` 분기 제거
- `case parser.SeqCall:` — `seq.Package`가 비어있으면 ERROR (`@func`에 `.` 필수)

### generator/go_templates.go
- `{{define "password"}}` 템플릿 제거
- `{{define "call_func"}}` 템플릿 수정 — `h.Pkg.Func(pkg.FuncInput{...})` 패턴

### generator/go_target.go
- **254행**: `SeqPassword` 분기 제거
- **451행**: `SeqPassword` import 분기 제거 (bcrypt 자동 추가 제거)
- **542행**: `SeqPassword` 기본 메시지 분기 제거
- **call 코드젠**: `seq.Package` + `seq.Func` → `h.Pkg.Func(pkg.FuncNameInput{...})` 생성
- **Handler struct 생성**: `func(...)` 필드 대신 `*pkg.Service` 필드 (패키지당 중복 제거)
- **import**: SSaC 스펙 파일의 import 선언에서 패키지 경로를 읽어 생성 코드에 그대로 옮김
