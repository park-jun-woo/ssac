# 수정지시서 008 실행결과: List 핸들러에서 ParseQueryOpts 직접 생성

## 실행일: 2026-03-13

## 방침

수동 파싱 코드(`c.Query("limit")`, `strconv.Atoi`, `allowedSort` map 등) 대신 `model.ParseQueryOpts(c, model.QueryOptsConfig{...})` 호출을 SSaC가 직접 생성하도록 변경.

## 변경 내용

### 1. `generateQueryOptsCode` 재작성 (`generator/go_helpers.go`)

기존: `opts := QueryOpts{}` + 수동 파싱 코드 생성
변경: `opts := model.ParseQueryOpts(c, model.QueryOptsConfig{...})` 생성

OpenAPI x-extensions 데이터를 config 리터럴로 변환:
- `x-pagination` → `Pagination: &model.PaginationConfig{Style, DefaultLimit, MaxLimit}`
- `x-sort` → `Sort: &model.SortConfig{Allowed, Default, Direction}`
- `x-filter` → `Filter: &model.FilterConfig{Allowed}`

각 확장이 없으면 해당 필드 생략 (nil).

### 2. `collectImports` 수정 (`generator/go_params.go`)

`needsQO`가 true일 때 `"strconv"` 대신 `"model"` import 추가. `model.ParseQueryOpts` 호출에 필요.

### 3. 테스트 업데이트 (`generator/go_handler_test.go`)

- `TestGenerateQueryArg`: `opts := QueryOpts{}` → `model.ParseQueryOpts(c, model.QueryOptsConfig{` 검증
- `TestGenerateSortAllowlist`: `allowedSort` map 패턴 → `&model.SortConfig{` + Allowed 배열 검증
- `TestGenerateSortNoXSort`: `allowedSort` 미포함 → `SortConfig` 미포함 검증

## 변경 파일

| 파일 | 변경 |
|---|---|
| `generator/go_helpers.go` | `generateQueryOptsCode` 재작성 |
| `generator/go_params.go` | `collectImports`에서 needsQO → `"model"` import |
| `generator/go_handler_test.go` | 3개 테스트 assertion 업데이트 |

## 생성 코드 예시

```go
func (h *Handler) ListGigs(c *gin.Context) {
    opts := model.ParseQueryOpts(c, model.QueryOptsConfig{
        Pagination: &model.PaginationConfig{Style: "offset", DefaultLimit: 20, MaxLimit: 100},
        Sort:       &model.SortConfig{Allowed: []string{"created_at", "title", "price"}, Default: "created_at"},
        Filter:     &model.FilterConfig{Allowed: []string{"status", "budget"}},
    })
    // ...
}
```
