# 수정지시서 010: validateResponse 및 ResponseFields 제거

## 배경

SSaC validator가 `@response` 필드 ↔ OpenAPI response schema 교차 검증을 수행하고 있다.
이 검증은 fullend `crosscheck/ssac_openapi.go`에서 이미 동일하게 수행 중이다 (Rule 5, Rule 6).

SSaC는 OpenAPI response 필드에 일절 관여하면 안 된다. SSOT 경계 원칙:
- SSaC 소관: 서비스 흐름 선언, guard 에러 코드
- fullend 소관: SSOT 간 교차 검증 (SSaC ↔ OpenAPI)

또한 현재 SSaC의 `loadOpenAPI`가 `op.Responses["200"]`만 하드코딩하여 201/204 응답을 읽지 못하는 버그도 있다.

## 변경 내용

### 1. `validator/validator.go` — `validateResponse` 함수 삭제

`validateResponse` 함수 전체 삭제 (약 474~538행).

`ValidateWithSymbols`에서 호출 제거:
```go
// 삭제:
errs = append(errs, validateResponse(sf, st)...)
```

`resolveWrapperFields` 함수도 `validateResponse`에서만 사용된다면 함께 삭제.

### 2. `validator/symbol.go` — `ResponseFields` 제거

`OperationSymbol` struct에서 `ResponseFields` 필드 삭제:
```go
// 삭제:
ResponseFields map[string]bool
```

`loadOpenAPI`에서 response fields 수집 코드 삭제 (372~380행):
```go
// 삭제:
// response fields (200)
if resp, ok := op.Responses["200"]; ok {
    ...
}
```

`OperationSymbol` 초기화에서 `ResponseFields: make(map[string]bool)` 삭제.

### 3. `validator/validator_test.go` — 관련 테스트 삭제

`ResponseFields`를 설정하는 테스트 케이스 전부 삭제 또는 해당 필드 제거.
`validateResponse` 관련 테스트 함수 삭제.

## 영향 없는 범위

- `validateRequest` (요청 필드 검증) — 유지
- `validateModel` (모델 존재 검증) — 유지
- `validateQueryUsage`, `validatePaginationType`, `validateCallInputTypes` — 유지
