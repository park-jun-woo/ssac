# 수정지시서 012 실행결과: @auth InputFields에서 UserID, Role 제외

## 실행일: 2026-03-13

## 방침

수정지시서011에서 `@auth` 템플릿에 `UserID`, `Role`을 고정 추가했으나, 사용자가 `.ssac` 파일에 `UserID: currentUser.ID`를 직접 작성하면 `InputFields`에도 포함되어 Go 컴파일 에러(`duplicate field name`) 발생. `@auth`의 `InputFields` 빌드 시 `UserID`, `Role` 키를 필터링.

## 변경 내용

### 1. `generator/go_helpers.go` — @auth InputFields 필터링

`buildTemplateData`에서 `@auth` 시퀀스의 inputs를 `InputFields`로 변환할 때, `UserID`와 `Role` 키를 제외.

### 2. `generator/go_handler_test.go` — 중복 방지 검증 추가

`TestGenerateAuthClaims`에 `strings.Count(code, "UserID:")` == 1 assertion 추가. inputs에 `UserID: currentUser.ID`가 명시되어도 생성 코드에 1번만 출력되는지 검증.

## 변경 파일

| 파일 | 변경 |
|---|---|
| `generator/go_helpers.go` | `@auth` InputFields에서 UserID, Role 키 필터링 |
| `generator/go_handler_test.go` | 중복 방지 assertion 추가 |
