# 수정지시서 012: @auth InputFields에서 UserID, Role 제외

## 배경

수정지시서 011에서 `@auth` 템플릿에 `UserID: currentUser.ID, Role: currentUser.Role`을 고정 추가했다.

그러나 SSaC 소스에서 `@auth` inputs에 `UserID: currentUser.ID`를 명시적으로 작성하는 경우, `InputFields`에도 `UserID`가 포함되어 **중복 필드**가 발생한다:

```go
authz.Check(authz.CheckRequest{
    Action: "PublishGig", Resource: "gig",
    UserID: currentUser.ID, Role: currentUser.Role,  // ← 템플릿 고정
    ResourceID: gig.ID, UserID: currentUser.ID,       // ← InputFields (중복!)
})
```

Go 컴파일 에러: `duplicate field name`.

## 변경 내용

### 1. `generator/go_helpers.go` — @auth InputFields 빌드 시 UserID, Role 필터링

`@auth` 시퀀스의 `InputFields` 구성 시, `seq.Inputs`에서 `UserID`와 `Role` 키를 제외한다. 이 두 필드는 템플릿에서 고정 출력하므로 `InputFields`에 포함되면 안 된다.

```go
// Inputs → InputFields (for state, auth, call)
if seq.Type == parser.SeqState || seq.Type == parser.SeqAuth || seq.Type == parser.SeqCall {
    if len(seq.Inputs) > 0 {
        inputs := seq.Inputs
        if seq.Type == parser.SeqAuth {
            // UserID, Role은 템플릿에서 고정 출력 — 중복 방지
            filtered := make(map[string]string)
            for k, v := range inputs {
                if k != "UserID" && k != "Role" {
                    filtered[k] = v
                }
            }
            inputs = filtered
        }
        d.InputFields = buildInputFieldsFromMap(inputs)
    }
}
```

### 2. 테스트 확인

`@auth` inputs에 `UserID`가 있는 케이스에서 생성 코드에 `UserID`가 1번만 나오는지 검증.
