# 수정지시서 011: @auth 템플릿에 UserID, Role 추가

## 배경

`@auth` 시퀀스가 `authz.Check(authz.CheckRequest{...})`를 생성할 때, `UserID`와 `Role`을 포함하지 않는다. `pkg/authz`의 `Check()` 함수는 `req.UserID`와 `req.Role`을 OPA input의 claims로 전달하므로, 이 값이 없으면 항상 UserID=0, Role=""로 평가된다.

`@auth`가 있는 핸들러는 반드시 JWT 미들웨어를 거치며, `currentUser`가 gin context에 주입되어 있다. SSaC 코드젠이 이미 `currentUser`를 context에서 추출하는 코드를 생성하므로, `@auth` 템플릿에서 고정으로 `UserID: currentUser.ID, Role: currentUser.Role`을 추가하면 된다.

## 변경 내용

### 1. `generator/go_templates.go` — `auth` 템플릿 2곳 수정

현재:
```
{{- define "auth" -}}
	if _, err {{if .FirstErr}}:={{else}}={{end}} authz.Check(authz.CheckRequest{Action: "{{.Action}}", Resource: "{{.Resource}}", {{.InputFields}} }); err != nil {
```

변경:
```
{{- define "auth" -}}
	if _, err {{if .FirstErr}}:={{else}}={{end}} authz.Check(authz.CheckRequest{Action: "{{.Action}}", Resource: "{{.Resource}}", UserID: currentUser.ID, Role: currentUser.Role, {{.InputFields}} }); err != nil {
```

파일 하단의 domain용 `auth` 템플릿(약 157행)도 동일하게 수정.

### 2. `generator/go_handler_test.go` — @auth 관련 테스트 assertion 수정

`authz.CheckRequest{`를 검증하는 assertion에 `UserID: currentUser.ID, Role: currentUser.Role` 포함 확인.

## 영향 없는 범위

- `@auth` 이외의 시퀀스 — 변경 없음
- `InputFields` 구성 로직 — 변경 없음 (ResourceID 등 기존 필드 그대로)
- `currentUser` context 추출 코드 — 이미 생성되고 있으므로 변경 없음
