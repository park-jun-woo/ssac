# 수정지시서 013 실행결과: DDL Index UNIQUE 제약 + PRIMARY KEY 추적

## 실행일: 2026-03-13

## 방침

`Index` struct에 `IsUnique` 필드, `DDLTable`에 `PrimaryKey` 필드 추가. DDL 파싱에서 인라인/독립 PRIMARY KEY, 인라인/독립 UNIQUE, CREATE UNIQUE INDEX 모두 추적.

## 변경 내용

### 1. `validator/symbol.go` — struct 필드 추가

- `DDLTable.PrimaryKey []string` — PK 컬럼명 목록
- `Index.IsUnique bool` — UNIQUE 인덱스 여부

### 2. `validator/symbol.go` — `parseDDLTables` 수정

- `PRIMARY KEY (col1, col2)` 독립 라인 → `extractParenColumns`로 PK 추출
- `UNIQUE (col1, col2)` 독립 라인 → IsUnique=true 인덱스 추가
- 인라인 `PRIMARY KEY` (`id BIGSERIAL PRIMARY KEY`) → `PrimaryKey` 추가
- 인라인 `UNIQUE` (`email VARCHAR UNIQUE`) → IsUnique=true 단일 컬럼 인덱스 추가

### 3. `validator/symbol.go` — `parseCreateIndex` 수정

- `CREATE UNIQUE INDEX` 감지 → `IsUnique: true` 설정

### 4. `validator/symbol.go` — `extractParenColumns` 헬퍼 추가

괄호 안 컬럼명 추출 (PRIMARY KEY, UNIQUE 공용).

### 5. `validator/validator_test.go` — 테스트 6개 추가

- `TestDDLPrimaryKey` — 인라인 PK
- `TestDDLPrimaryKeyComposite` — 복합 PK
- `TestDDLInlineUnique` — 인라인 UNIQUE
- `TestDDLUniqueConstraint` — 독립 UNIQUE 제약
- `TestDDLCreateUniqueIndex` — CREATE UNIQUE INDEX
- `TestDDLCreateNonUniqueIndex` — 일반 INDEX는 IsUnique=false
