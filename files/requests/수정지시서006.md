# 수정지시서 006: FK 참조 엔티티 nil 체크 자동 생성

## 발신
fullend (BUG010)

## 심각도
MEDIUM

## 현상

SSaC codegen이 FK 참조로 엔티티를 조회한 후(`FindByID`) nil 체크 없이 바로 필드에 접근하여, 참조 대상이 삭제된 경우 nil pointer dereference 패닉이 발생한다.

### 재현 예시

SSaC `.ssac` 파일:
```
@get schedule  → schedule
@get project by schedule.project_id  → project
@auth update_schedule project
```

생성된 코드 (`update_schedule.go`):
```go
schedule, err := h.ScheduleModel.FindByID(id)
if err != nil { ... }

project, err := h.ProjectModel.FindByID(schedule.ProjectID)
if err != nil { ... }
// project가 nil일 수 있음 (FK 대상 삭제 후 고아 레코드)

if err := authz.Check(authz.CheckRequest{
    ResourceID: project.ID,  // ← nil pointer dereference 패닉
}); err != nil { ... }
```

## 기대 동작

FK 참조로 조회한 엔티티(`@get X by Y.fk_id`)에 대해, 조회 결과가 nil인 경우 404를 반환하는 코드를 자동 생성해야 한다.

```go
project, err := h.ProjectModel.FindByID(schedule.ProjectID)
if err != nil { ... }
if project == nil {                                    // ← 추가
    c.JSON(http.StatusNotFound, gin.H{"error": "project not found"})
    return
}
```

## 영향 범위

`@get X by Y.fk_id` 패턴을 사용하는 모든 핸들러. 예:
- `update_schedule.go` — `project` nil 체크 누락
- `delete_schedule.go` — 동일

## 수정 제안

SSaC codegen에서 `@get` 시퀀스가 FK 참조 조회(`by other.fk_id`)인 경우, `FindByID` 호출 직후 nil 체크 + 404 반환 코드를 자동 삽입.

판별 기준: `@get` 시퀀스의 소스가 다른 변수의 필드(`schedule.project_id`)이면 FK 참조 조회로 간주.
