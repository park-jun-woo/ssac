# 수정지시서 011 실행결과: @auth 템플릿에 UserID, Role 고정 추가

## 실행일: 2026-03-13

## 방침

`@auth` 템플릿에 `UserID: currentUser.ID, Role: currentUser.Role`을 고정으로 포함. 기존 `go_helpers.go`의 조건부 자동 추가 로직은 제거.

## 변경 내용

### 1. `generator/go_templates.go` — auth 템플릿 2곳

- `auth` (HTTP): `authz.CheckRequest{...}` 에 `UserID: currentUser.ID, Role: currentUser.Role,` 고정 추가
- `sub_auth` (subscribe): 동일

### 2. `generator/go_helpers.go` — needsCurrentUser 수정

`@auth` 시퀀스가 있으면 항상 `needsCurrentUser = true` 반환. `currentUser` context 추출 코드가 항상 생성됨.

기존 `@auth + currentUser 참조 → Role, UserID 자동 추가` 조건부 로직 제거 (템플릿에 고정이므로 불필요).

### 3. `generator/go_args.go` — 미사용 함수 삭제

- `hasCurrentUserRef` 삭제
- `copyInputs` 삭제

### 4. `generator/go_handler_test.go` — 테스트 2개 수정

- `TestGenerateAuthNoCurrentUser` → `TestGenerateAuthAlwaysHasCurrentUser`: `currentUser` 추출 + UserID, Role 포함 검증
- `TestGenerateAuthNoClaimsWithoutCurrentUser` → `TestGenerateAuthAlwaysIncludesUserIDRole`: UserID, Role 항상 포함 검증

## 변경 파일

| 파일 | 변경 |
|---|---|
| `generator/go_templates.go` | `auth`, `sub_auth` 템플릿에 UserID, Role 고정 추가 |

| `generator/go_handler_test.go` | 테스트 2개 수정 |
