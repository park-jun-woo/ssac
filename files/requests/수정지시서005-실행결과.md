# 수정지시서 005 실행결과: @call 에러 코드 — 대상 함수의 @error 어노테이션 읽기

## 실행일: 2026-03-13

## 변경 내용

### 1. `MethodInfo.ErrStatus` 필드 추가

`validator/symbol.go`의 `MethodInfo` 구조체에 `ErrStatus int` 필드 추가. `@error` 어노테이션 값을 저장 (0이면 미지정).

### 2. `loadPackageGoInterfaces` 3차 스캔 추가

기존 1차(Request struct), 2차(interface) 스캔에 3차 standalone function 스캔 추가:

- `func Xxx(req XxxRequest)` 패턴의 standalone function 탐색
- doc comment에서 `// @error NNN` 파싱
- `"pkgName._func"` 키로 `st.Models`에 등록 (interface 모델과 충돌 방지)

### 3. `@call` ErrStatus 3단계 우선순위

`generator/go_helpers.go`에서 `@call` ErrStatus 결정 로직 변경:

```
1. .ssac 파일 명시값 (seq.ErrStatus)     → 최우선
2. SymbolTable @error 어노테이션          → 중간
3. 기본값                                 → 500
```

`lookupCallErrStatus` 헬퍼 함수 추가: `st.Models`에서 `pkgName.` 접두사 매칭 → 해당 함수의 `ErrStatus` 반환.

### 생성 코드 변경

```go
// 대상 함수에 @error 401 어노테이션, .ssac에 명시 없음
// 변경 전: http.StatusInternalServerError (500)
// 변경 후: http.StatusUnauthorized (401)
if _, err = auth.VerifyPassword(auth.VerifyPasswordRequest{...}); err != nil {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "호출 실패"})
    return
}
```
