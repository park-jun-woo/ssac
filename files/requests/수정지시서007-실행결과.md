# 수정지시서 007 실행결과: ssac-gen 미사용 import 제거

## 실행일: 2026-03-13

## 방침

방법 A (권장) 채택 — 함수 본문을 먼저 생성한 뒤, 실제 패키지 참조를 스캔하여 import를 필터링.

## 원인 분석

- `"database/sql"` — `h.DB.BeginTx()` 호출은 Handler struct 타입 추론으로 동작. per-function 파일에서 `sql.` 직접 참조 없음. `database/sql`은 handler.go에만 필요.
- `"strconv"` — path param이나 QueryOpts가 없는 함수에서도 후보 import에 포함되는 경우 존재.

## 변경 내용

### 1. `filterUsedImports` 헬퍼 추가

`generator/go_helpers.go`에 추가. import 경로의 마지막 세그먼트(패키지명)가 본문에 `pkg.` 형태로 참조되는지 스캔.

```go
func filterUsedImports(imports []string, body string) []string {
    var used []string
    for _, imp := range imports {
        pkg := imp
        if idx := strings.LastIndex(imp, "/"); idx >= 0 {
            pkg = imp[idx+1:]
        }
        if strings.Contains(body, pkg+".") {
            used = append(used, imp)
        }
    }
    return used
}
```

### 2. `generateHTTPFunc` / `generateSubscribeFunc` 구조 변경

기존: package → imports → body 순차 생성
변경: body 먼저 생성 → 후보 import 수집 → `filterUsedImports`로 필터 → 최종 조립

두 함수 모두 동일 패턴 적용.
