# 수정지시서 006 실행결과: FK 참조 엔티티 nil 체크 — validator ERROR

## 실행일: 2026-03-13

## 방침

codegen 자동 삽입(A안) 대신 **validator ERROR(B안)** 채택. SSaC의 "명시적 선언" 원칙을 유지하면서 nil pointer dereference를 방지한다.

- FK 참조 `@get` 후 `@empty` 가드가 없으면 → **ERROR** (코드 생성 중단)
- `@get!`로 억제 불가 (ERROR는 `!` 접미사로 억제할 수 없음)

## 변경 내용

### 1. `validateFKReferenceGuard` 함수 추가

`validator/validator.go`에 FK 참조 `@get` 후 `@empty` 가드 누락을 검증하는 함수 추가.

**FK 참조 판별 기준**:
- `@get` 시퀀스의 input value가 이전 result 변수의 필드를 참조 (`schedule.ProjectID`)
- `request.*`, `currentUser.*`, `query`, `message.*`, `config.*`, 리터럴은 제외

**스킵 조건**:
- `[]Type` (슬라이스 결과) — empty slice, nil dereference 위험 없음
- `Page[T]`/`Cursor[T]` (래퍼 타입) — wrapper struct
**에러 메시지**:
```
ERROR: "project" — FK 참조 조회 후 @empty 가드가 필요합니다
```

### 2. `validateFunc`에 호출 추가

`validateFunc` 함수에서 `validateFKReferenceGuard` 호출 추가.
