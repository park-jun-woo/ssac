# 수정지시서 003 실행결과: 쓰기 시퀀스 자동 트랜잭션 래핑

## 실행일: 2026-03-13

## 변경 내용

### 1. 쓰기 감지 — `hasWriteSequence` 헬퍼

`generator/go_helpers.go`에 `hasWriteSequence()` 추가. 시퀀스 목록에서 `@post`/`@put`/`@delete` 존재 여부 판단.

### 2. Handler struct + DB 주입

`generator/go_target.go`에 `GenerateHandlerStruct()` 구현. 도메인별 `handler.go` 생성:

```go
type Handler struct {
    DB             *sql.DB
    ProposalModel  model.ProposalModel
    GigModel       model.GigModel
}
```

- `collectDomainModels()`: 도메인별 사용 모델 수집 (`@call` 제외)
- `target.go` Target 인터페이스에 `GenerateHandlerStruct` 메서드 추가
- `generator.go`에 top-level 래퍼 함수 추가

### 3. 함수 시그니처 변경

**HTTP 핸들러:**
```go
// 변경 전: func Name(c *gin.Context)
// 변경 후: func (h *Handler) Name(c *gin.Context)
```

**Subscribe 핸들러:**
```go
// 변경 전: func Name(ctx context.Context, message T) error
// 변경 후: func (h *Handler) Name(ctx context.Context, message T) error
```

### 4. 모델 호출 패턴 변경

```go
// 변경 전 (package-level 변수)
courseModel.FindByID(id)

// 변경 후 (Handler struct DI)
h.CourseModel.FindByID(id)              // 읽기 전용 함수
h.CourseModel.WithTx(tx).FindByID(id)   // 트랜잭션 함수 (읽기+쓰기 모두)
```

### 5. 트랜잭션 자동 래핑

쓰기 시퀀스가 있는 함수에 자동 삽입:

**HTTP:**
```go
tx, err := h.DB.BeginTx(c.Request.Context(), nil)
if err != nil {
    c.JSON(http.StatusInternalServerError, gin.H{"error": "transaction failed"})
    return
}
defer tx.Rollback()

// ... 시퀀스 ...

if err = tx.Commit(); err != nil {
    c.JSON(http.StatusInternalServerError, gin.H{"error": "commit failed"})
    return
}
c.JSON(200, gin.H{...})
```

**Subscribe:**
```go
tx, err := h.DB.BeginTx(ctx, nil)
if err != nil {
    return fmt.Errorf("transaction failed: %w", err)
}
defer tx.Rollback()

// ... 시퀀스 ...

if err = tx.Commit(); err != nil {
    return fmt.Errorf("commit failed: %w", err)
}
return nil
```

### 6. Model interface에 WithTx 추가

`generator/go_interface.go`에서 모든 모델 인터페이스에 `WithTx` 메서드 자동 생성:

```go
type CourseModel interface {
    WithTx(tx *sql.Tx) CourseModel
    FindByID(courseID int64) (*Course, error)
    // ...
}
```

`database/sql` import 무조건 추가 (모든 인터페이스에 WithTx 있으므로).

### 7. errDeclared 연동

`BeginTx`가 `err`을 선언하므로 tx 함수에서 `errDeclared = true` 시작. 이후 시퀀스에서 `:=` vs `=` 정확히 추적.

### 8. Commit 배치

- `@response` 직전에 Commit 삽입 (HTTP)
- `return nil` 직전에 Commit 삽입 (Subscribe)
- `@response` 없는 함수: 시퀀스 루프 후 fallback Commit (`committed` 플래그로 중복 방지)

### 9. import 추가

`generator/go_params.go`: 쓰기 시퀀스 있으면 `"database/sql"` import 추가 (HTTP + Subscribe 모두).

## 설계 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| DB 참조 방식 | Handler struct (Option B) | 업계 정석 DI 패턴, 기술 부채 없음 |
| `@call` tx 포함 | 제외 | `@call`은 외부 로직, 리소스 접근 불가 (계약) |
| `@publish` tx 포함 | 포함 | 사용자 설계 의도 보존, 순서 변경 금지 |
| Subscribe tx | 쓰기 있으면 적용 | HTTP와 동일 규칙 |
| 트랜잭션 범위 | 읽기+쓰기 전체 | 읽기 정합성도 보장 |
