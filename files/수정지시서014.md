# 수정지시서 014: @component 폐지 — @func만 허용

## 배경

`@sequence call`에서 `@component`와 `@func` 두 가지를 허용해왔으나, `@component`는 다음 문제가 있다:

1. **구현체 부재** — interface만 선언하고 실제 구현체가 없으면 런타임 nil panic 발생
2. **주입 경로 불명확** — main.go에서 누가 구현체를 생성·주입하는지 codegen이 보장할 수 없음
3. **@func과 역할 중복** — 외부 로직 호출은 `@func pkg.funcName`으로 통일 가능
4. **notification 같은 복잡 기능은 @func 하나로도 안 됨** — 별도 인프라(메시지 큐, 외부 서비스) 필요. SSOT 범위 밖

fullend 측에서는 `@component` 사용 시 ERROR를 발생시키기로 결정.
ssac 측에서도 파서·검증·코드젠에서 `@component`를 완전 제거해야 한다.

## 변경 요청

### 1. parser/types.go
- `Sequence.Component` 필드 삭제

### 2. parser/parser.go
- `case "component":` 분기 삭제 (또는 파싱 시 에러 반환)

### 3. validator/validator.go
- `validateCallTarget`: `@component` 관련 검증 로직 삭제
- `@sequence call`에서 `@func` 필수, `@component` 사용 시 ERROR:
  - `"@component는 폐지되었습니다. @func pkg.funcName을 사용하세요"`

### 4. validator/symbol.go
- `SymbolTable.Components` 맵이 있다면 삭제

### 5. generator/go_target.go
- `call_component` 분기 삭제 (seq.Component != "" 분기 제거)

### 6. generator/go_templates.go
- `{{define "call_component"}}` 템플릿 삭제

### 7. 테스트 파일
- `parser/parser_test.go`: `@component` 테스트 케이스 삭제 또는 에러 기대로 변경
- `validator/validator_test.go`: 동일
- `testdata/backend-service/delete_project.go`: `@component` 사용 시 제거

### 8. ssac 내 dummy 스펙
- `specs/dummy-study/service/cancel_reservation.go`: `@component notification` 시퀀스 삭제
- `specs/dummy-study/service/create_reservation.go`: 동일
- `specs/dummy-study/model/notification.go`: 파일 삭제

## 검증 방법

```bash
# 1. @component 사용 시 에러 확인
echo '// @sequence call
// @component notification
// @param enrollment' > /tmp/test-component.go
ssac validate /tmp/test-component.go  # → ERROR: @component는 폐지되었습니다

# 2. 기존 테스트 통과
go test ./...

# 3. @func만 사용하는 스펙은 정상 통과
ssac validate specs/dummy-study
```
