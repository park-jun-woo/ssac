# 수정지시서 008: guard state 시퀀스 타입 추가

## 요청자
fullend (Phase020-StateTransition)

## 요약

SSaC에 11번째 시퀀스 타입 `guard state`를 추가한다. 리소스의 상태 전이 가능 여부를 검사하는 시퀀스다.

## 문법

```go
// @sequence guard state {stateDiagramID}
// @param {entity}.{StatusField}
```

- `{stateDiagramID}`: Mermaid stateDiagram 파일 식별자 (예: `course`)
- `{entity}`: 이전 `@result`로 바인딩된 변수명
- `{StatusField}`: 상태를 나타내는 필드명 (DDL 컬럼에 매핑)
- **함수명이 전이 이벤트로 사용됨**: `PublishCourse` 함수 안의 `guard state course`는 현재 상태에서 `PublishCourse` 전이가 유효한지 검사

## 사용 예시

```go
// @sequence get
// @model Course.FindByID
// @param CourseID request
// @result course Course
//
// @sequence guard nil course
// @message "강의를 찾을 수 없습니다"
//
// @sequence guard state course
// @param course.Published
//
// @sequence put
// @model Course.Publish
// @param CourseID request
//
// @sequence response json
func PublishCourse(w http.ResponseWriter, r *http.Request) {}
```

## 코드젠 출력

```go
// guard state: course (PublishCourse)
if !coursestate.CanTransition(course.Published, "PublishCourse") {
    http.Error(w, "invalid state transition", http.StatusConflict)
    return
}
```

- import: `"{module}/states/coursestate"` (stateDiagramID + "state" 패키지)
- 상태 머신 패키지 자체는 fullend가 생성함. SSaC는 호출 코드만 생성.

## 수정 범위

### 1. parser

`guard state`를 파싱한다. 기존 `guard nil`/`guard exists`와 패턴이 유사하다.

```
guard nil    → SequenceType: "guard nil",    Target: "course"
guard exists → SequenceType: "guard exists", Target: "existing"
guard state  → SequenceType: "guard state",  Target: "course" (= stateDiagramID)
```

`guard state`의 `@param`은 `entity.Field` 형태 (dot 포함). 기존 `@param`과 달리 source가 없고 단일 값이다.

파싱 결과 구조:

```go
// Sequence.Type = "guard state"
// Sequence.Target = "course"              (stateDiagramID)
// Sequence.Params[0].Name = "course"      (entity)
// Sequence.Params[0].Source = "Published"  (StatusField)
// 또는 Sequence.StateParam = "course.Published" (전용 필드)
```

구현 방식은 SSaC 내부 구조에 맞게 결정. 핵심은:
- `guard state` 뒤의 토큰 → stateDiagramID (= Target)
- `@param` 뒤의 `entity.Field` → dot split하여 entity + field 추출

### 2. validator

`guard state` 검증 규칙:

| 규칙 | 수준 |
|---|---|
| `@param` 형태가 `entity.Field`인지 (dot 포함) | ERROR |
| `entity`가 이전 `@result` 변수로 존재하는지 | ERROR |
| `@param`이 정확히 1개인지 | ERROR |

stateDiagramID가 실제 존재하는지는 fullend 교차 검증에서 확인한다. SSaC 단독으로는 검증하지 않는다.

### 3. generator

코드젠 템플릿:

```go
// guard state 시퀀스
if !{{.Target}}state.CanTransition({{.Entity}}.{{.StatusField}}, "{{.FuncName}}") {
    http.Error(w, "invalid state transition", http.StatusConflict)
    return
}
```

- `{{.Target}}`: stateDiagramID (예: `course`)
- `{{.Entity}}`: @param의 entity 부분 (예: `course`)
- `{{.StatusField}}`: @param의 field 부분 (예: `Published`)
- `{{.FuncName}}`: 현재 SSaC 함수명 (예: `PublishCourse`)
- import 추가: `"{module}/states/{{.Target}}state"`

### 4. 매뉴얼 업데이트

`artifacts/manual-for-ai.md`와 `artifacts/manual-for-human.md`:
- 시퀀스 타입 테이블: 10개 → 11개
- `guard state` 행 추가: `guard state | 상태 전이 검사 | @param (entity.Field)`
- SSaC 닫힌 집합 언급 부분 갱신: "10가지" → "11가지"

## 기존 코드 영향

- `guard nil`, `guard exists`와 같은 guard 계열이므로 파서 구조 자연스럽게 확장
- 기존 10개 시퀀스의 동작 변경 없음
- 새 시퀀스 추가만으로 하위 호환 유지
