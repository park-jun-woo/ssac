# 수정지시서 008 실행결과

## 1. parser: guard state 파싱 — 완료

`SeqGuardState = "guard state"` 상수 추가, `ValidSequenceTypes`에 등록.

기존 `guard nil`/`guard exists`와 동일한 2단어 패턴이므로 `parseSequenceType`은 자동 처리. `parseGuardTarget` 호출 조건에 `SeqGuardState` 추가하여 3번째 단어(stateDiagramID)를 Target으로 추출.

파싱 결과:
```
// @sequence guard state course
// @param course.Published

Sequence.Type = "guard state"
Sequence.Target = "course"                  // stateDiagramID
Sequence.Params[0].Name = "course.Published" // entity.Field
```

## 2. validator: guard state 검증 — 완료

`validateRequiredFields`에 guard state 분기 추가:
- Target 필수
- `@param` 정확히 1개
- `@param`에 dot 포함 (entity.Field 형식)

`validateVariableFlow`에서 `guard state`의 Target은 stateDiagramID이므로 변수 참조 검증에서 제외. `@param`의 entity 부분은 기존 `paramVarRef` 로직으로 미선언 변수 검증.

## 3. generator: guard state 코드젠 — 완료

`templateData`에 `Entity`, `StatusField`, `FuncName` 필드 추가.

`buildTemplateData`에서 `@param`의 dot notation을 split하여 Entity/StatusField 설정.

생성 코드:
```go
// guard state
if !coursestate.CanTransition(course.Published, "PublishCourse") {
    http.Error(w, "상태 전이가 허용되지 않습니다", http.StatusConflict)
    return
}
```

import: `"states/coursestate"` (placeholder — fullend가 module prefix 후처리).

`collectImports`에서 동적 import 처리를 위해 고정 순서 리스트 이후 남은 seen 항목도 추가하도록 변경.
