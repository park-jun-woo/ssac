# 수정지시서 004: Generator Target 추상화 — 다중 언어 코드젠 슬롯

## 배경

fullend는 현재 Go 백엔드만 생성하지만, 장기적으로 Java, PHP, Node.js 등 다양한 언어를 지원해야 한다.
SSaC의 parser가 생성하는 IR (`ServiceFunc`, `Sequence`)은 이미 언어 중립적이다.
그러나 generator는 Go 코드를 직접 string-build + text/template으로 생성하므로, 다른 언어를 추가하려면 generator 전체를 복사해야 한다.

**목표:** generator를 Target 인터페이스 기반으로 재구조화하여, 새 언어를 추가할 때 해당 Target만 구현하면 되는 구조를 만든다.

---

## 1. Target 인터페이스 정의

### 현재

```go
// generator/generator.go
func Generate(funcs []parser.ServiceFunc, outDir string, st *validator.SymbolTable) error
func GenerateFunc(sf parser.ServiceFunc, st *validator.SymbolTable) ([]byte, error)
func GenerateModelInterfaces(funcs []parser.ServiceFunc, st *validator.SymbolTable, outDir string) error
```

모든 함수가 Go 코드를 직접 생성한다. 언어 선택 여지가 없다.

### 변경

```go
// generator/target.go  ★ NEW
package generator

import (
    "github.com/geul-org/ssac/parser"
    "github.com/geul-org/ssac/validator"
)

// Target은 특정 언어의 코드 생성기가 구현해야 하는 인터페이스이다.
type Target interface {
    // GenerateFunc는 하나의 서비스 함수를 타겟 언어 소스 코드로 변환한다.
    GenerateFunc(sf parser.ServiceFunc, st *validator.SymbolTable) ([]byte, error)

    // GenerateModelInterfaces는 서비스 함수에서 사용하는 모델 인터페이스를 생성한다.
    GenerateModelInterfaces(funcs []parser.ServiceFunc, st *validator.SymbolTable, outDir string) error

    // FileExtension은 생성 파일의 확장자를 반환한다. (예: ".go", ".java", ".ts")
    FileExtension() string
}
```

### 기존 함수 시그니처 유지 (하위 호환)

```go
// generator/generator.go — 기존 시그니처 유지, 내부 위임
func Generate(funcs []parser.ServiceFunc, outDir string, st *validator.SymbolTable) error {
    return GenerateWith(DefaultTarget(), funcs, outDir, st)
}

func GenerateFunc(sf parser.ServiceFunc, st *validator.SymbolTable) ([]byte, error) {
    return DefaultTarget().GenerateFunc(sf, st)
}

func GenerateModelInterfaces(funcs []parser.ServiceFunc, st *validator.SymbolTable, outDir string) error {
    return DefaultTarget().GenerateModelInterfaces(funcs, st, outDir)
}

// GenerateWith는 지정된 Target으로 코드를 생성한다.
func GenerateWith(t Target, funcs []parser.ServiceFunc, outDir string, st *validator.SymbolTable) error {
    if err := os.MkdirAll(outDir, 0755); err != nil {
        return err
    }
    for _, sf := range funcs {
        src, err := t.GenerateFunc(sf, st)
        if err != nil {
            return fmt.Errorf("%s: %w", sf.FileName, err)
        }
        ext := t.FileExtension()
        outName := strings.TrimSuffix(sf.FileName, ".go") + ext
        outPath := filepath.Join(outDir, outName)
        if err := os.WriteFile(outPath, src, 0644); err != nil {
            return err
        }
    }
    return nil
}

// DefaultTarget은 Go Target을 반환한다.
func DefaultTarget() Target {
    return &GoTarget{}
}
```

## 2. Go Target 구현 — 기존 코드 이동

### 현재

generator.go에 Go 코드 생성 로직이 직접 작성되어 있다:
- `GenerateFunc()` — Go 함수 시그니처, import, package 선언, gofmt
- `generateExtractCode()` — `r.FormValue()`, `json.NewDecoder()`, `strconv.ParseInt()`
- `buildTemplateData()` — Go 변수명 규칙 (lcFirst 등)
- templates.go — Go text/template (`if err != nil`, `http.Error` 등)
- model_interface.go — Go interface 문법, Go 타입 (`int64`, `string`, `*Type`)

### 변경

기존 코드를 `GoTarget` struct의 메서드로 이동한다.

```
generator/
├── target.go           ★ NEW: Target 인터페이스 정의
├── generator.go        수정: 하위 호환 래퍼 + GenerateWith()
├── go_target.go        ★ RENAME: 기존 GenerateFunc/model 로직 이동
├── go_templates.go     ★ RENAME: templates.go → go_templates.go
└── generator_test.go   수정: 기존 테스트 유지 (API 불변)
```

```go
// generator/go_target.go  ★ NEW (기존 generator.go + model_interface.go 에서 이동)
package generator

type GoTarget struct{}

func (g *GoTarget) FileExtension() string { return ".go" }

func (g *GoTarget) GenerateFunc(sf parser.ServiceFunc, st *validator.SymbolTable) ([]byte, error) {
    // 기존 GenerateFunc() 본문 그대로
}

func (g *GoTarget) GenerateModelInterfaces(funcs []parser.ServiceFunc, st *validator.SymbolTable, outDir string) error {
    // 기존 GenerateModelInterfaces() 본문 그대로
}

// 이하 기존 private 함수들 (generateExtractCode, buildTemplateData, collectTypedRequestParams 등)
// GoTarget의 메서드 또는 패키지 내부 함수로 유지
```

### 이동 대상 함수 목록

| 현재 위치 | 이동 후 |
|---|---|
| `generator.go:GenerateFunc()` | `go_target.go:(*GoTarget).GenerateFunc()` |
| `generator.go:generateExtractCode()` | `go_target.go` 내부 함수 |
| `generator.go:buildTemplateData()` | `go_target.go` 내부 함수 |
| `generator.go:collectTypedRequestParams()` | `go_target.go` 내부 함수 |
| `generator.go:resolveParam()`, `resolveParamRef()` | `go_target.go` 내부 함수 |
| `generator.go:lookupDDLType()` | `go_target.go` 내부 함수 |
| `model_interface.go:GenerateModelInterfaces()` | `go_target.go:(*GoTarget).GenerateModelInterfaces()` |
| `model_interface.go:collectModelUsages()` 등 | `go_target.go` 내부 함수 |
| `templates.go` (전체) | `go_templates.go` (파일명만 변경) |

### 유지할 함수 (언어 중립)

| 함수 | 이유 |
|---|---|
| `Generate()` | 하위 호환 래퍼 |
| `GenerateWith()` | Target 기반 진입점 |
| `DefaultTarget()` | Go Target 팩토리 |
| `lcFirst()`, `toSnakeCase()` | 범용 유틸 (다른 Target에서도 사용) |

## 3. 차후 언어 추가 예시

새 언어 추가 시 해야 할 일:

```go
// generator/java_target.go (미래)
package generator

type JavaTarget struct{}

func (j *JavaTarget) FileExtension() string { return ".java" }

func (j *JavaTarget) GenerateFunc(sf parser.ServiceFunc, st *validator.SymbolTable) ([]byte, error) {
    // ServiceFunc IR → Java Spring Controller 메서드 생성
    // sf.Sequences를 순회하며 Java 코드 빌드
}

func (j *JavaTarget) GenerateModelInterfaces(funcs []parser.ServiceFunc, st *validator.SymbolTable, outDir string) error {
    // Java interface 생성
}
```

fullend 측 호출:

```go
// fullend orchestrator
target := generator.DefaultTarget() // Go
// 또는
// target := &generator.JavaTarget{} // Java

generator.GenerateWith(target, funcs, outDir, st)
```

## 4. Target 선택 메커니즘

당장은 `DefaultTarget() → GoTarget` 고정이다.
fullend가 Target을 선택하여 `GenerateWith()`에 전달하는 구조이므로, SSaC 내부에서 Target 선택 로직은 불필요하다.

Target 등록/발견:

```go
// generator/registry.go  ★ NEW (선택사항, 없어도 동작)
var targets = map[string]Target{
    "go": &GoTarget{},
}

func RegisterTarget(name string, t Target) {
    targets[name] = t
}

func GetTarget(name string) (Target, bool) {
    t, ok := targets[name]
    return t, ok
}
```

이것은 선택 사항이다. fullend가 직접 `&GoTarget{}`을 생성해도 된다.

## 변경 파일 목록

| 파일 | 변경 유형 |
|---|---|
| `generator/target.go` | ★ NEW: Target 인터페이스 |
| `generator/go_target.go` | ★ NEW: 기존 Go 생성 로직 이동 (generator.go + model_interface.go) |
| `generator/go_templates.go` | ★ RENAME: templates.go → go_templates.go |
| `generator/generator.go` | 수정: Generate/GenerateFunc → DefaultTarget 위임 + GenerateWith 추가 |
| `generator/model_interface.go` | 삭제 (go_target.go로 이동) |
| `generator/generator_test.go` | 수정 없음 (외부 API 불변이므로 기존 테스트 그대로 통과해야 함) |

## 하위 호환성

**외부 API 변경 없음:**
- `Generate()` — 시그니처 동일, 내부만 DefaultTarget 위임
- `GenerateFunc()` — 시그니처 동일
- `GenerateModelInterfaces()` — 시그니처 동일

**추가 API:**
- `GenerateWith(target Target, ...)` — 새 진입점
- `DefaultTarget() Target` — GoTarget 팩토리
- `Target` 인터페이스 — 새 타입

기존 fullend 코드는 수정 없이 동작한다.
fullend가 Target 선택 기능을 추가할 때 `GenerateWith()`로 전환하면 된다.

## 우선순위

**중간** — 현재 Go만 지원하므로 즉시 필요하지 않다. 그러나 구조 변경은 코드가 커지기 전에 하는 것이 비용이 적다.
기존 테스트가 모두 통과하면 위험도가 낮은 리팩토링이다.
