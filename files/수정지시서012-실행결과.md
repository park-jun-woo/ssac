## 수정 항목별 결과

### 1. import 경로: spec 파일의 import 선언을 사용 ✅

- `parser/types.go`: `ServiceFunc`에 `Imports []string` 필드 추가
- `parser/parser.go`: `collectImportsFromAST()` 함수 추가 — Go AST에서 import 경로 수집 (`"net/http"` 제외)
- `parser/parser.go`: `ParseFile()`에서 `sf.Imports = collectImportsFromAST(f)` 설정
- `generator/go_target.go`: `collectImports()`에서 `@func` 패키지 자동 추가 로직 제거, `specImports []string` 파라미터 추가하여 spec 파일 import를 그대로 병합
- `parser/parser_test.go`: `TestParseFileImports` 추가 — import 파싱 및 `"net/http"` 제외 검증

### 2. struct 이름: `*Input` → `*Request` ✅

- `generator/go_templates.go`: `call_func` 템플릿에서 `{{.FuncMethod}}Input` → `{{.FuncMethod}}Request`
- `generator/generator_test.go`: `TestGenerateDeleteProject` 기대값 업데이트

### 3. 결과 없는 call의 반환값 처리 ✅

- `generator/go_templates.go`: `call_func` 템플릿에서 `@result` 없을 때 `_` 사용
  - `{{if .Result}}out{{else}}_{{end}}, err {{if .FirstErr}}:={{else}}={{end}} ...`

### 4. 결과 추출 필드명: `@result var Type.Field` ✅

- `parser/types.go`: `Result` struct에 `Field string` 필드 추가
- `parser/parser.go`: `parseResult()` 확장 — `Type`에 `.`이 있으면 `Type`과 `Field`를 분리
  - `"token Token.AccessToken"` → `Var="token"`, `Type="Token"`, `Field="AccessToken"`
- `generator/go_target.go`: `buildTemplateData`에서 `seq.Result.Field`이 있으면 사용, 없으면 기존 `ucFirst(var)` 동작
- `generator/go_templates.go`: `call_func` 템플릿에서 `out.{{.ResultField}}` 사용
- `parser/parser_test.go`: `TestParseResult` 확장 — `Field` 파싱 검증

### 5. Input struct 필드명: `->` 매핑 지원 ✅

- `generator/go_target.go`: `buildInputFields()` 함수 추가
  - `p.Column`이 있으면 fieldName으로 사용 (기존 `->` 매핑 인프라 재활용)
  - `@param user.ID -> UserID` → `{UserID: user.ID}`
